# ad-mcc-36-执行闭环反馈单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-36 |
| 模块名称 | 执行闭环反馈单元 |
| 所属分区 | 十、执行反馈与日志 |
| 核心职责 | 汇总各执行模块（转向、动力、制动、姿态、灯光、档位等）每次操控动作的完整执行结果，包括偏差报告、完成信号、响应延迟与目标达成判定。将分散的偏差数据整合为统一的“动作闭环回执”，上报至 ad-mcc-01 供上层决策与元认知评估，同时推送至 ad-mcc-37 供运动质量评估使用。不参与任何操控决策，仅负责执行结果的汇总、比对与闭环反馈 |
| 依赖模块 | ad-mcc-07（转向执行偏差监控单元，提供转向偏差数据）、ad-mcc-12（动力执行偏差监控单元，提供动力偏差数据）、ad-mcc-16（制动执行偏差监控单元，提供制动偏差数据）、ad-mcc-18（车身姿态实时监测单元，提供姿态执行状态）、ad-mcc-22（转向灯自动控制单元，提供灯光执行状态）、ad-mcc-26（档位切换管控单元，提供档位执行状态） |
| 被依赖模块 | ad-mcc-01（小脑总控调度核心，接收动作闭环回执）、ad-mcc-37（运动质量评估单元，消费汇总的偏差数据）、ad-mcc-38（执行日志记录单元，记录闭环事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无待处理的操控动作，等待新指令 | 系统初始化完成，无动作指令 |
| 汇总收集中 | `COLLECTING` | 已收到 ad-mcc-01 下发的动作指令通知，正在等待各执行模块反馈 | ad-mcc-01 下发动作指令时 |
| 闭环判定中 | `JUDGING` | 所有相关执行模块已反馈，正在进行目标达成判定 | 收到全部模块反馈或超时 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 动作闭环回执 | Struct（指令ID + 目标达成判定 + 各维度偏差汇总 + 综合偏差评级 + 响应延迟最大值 + 在线模块数/总数） | ad-mcc-01（小脑总控调度核心） | 所有模块反馈完成或超时后 | **高** |
| 偏差数据汇总包 | Struct（指令ID + 各维度偏差明细 + 时间戳） | ad-mcc-37 运动质量评估单元 | 闭环回执生成后 | 普通 |
| 闭环事件记录 | Struct（指令ID + 目标达成 + 耗时 + 偏差摘要） | ad-mcc-38 执行日志记录单元 | 闭环回执生成后 | 普通 |

## 目标达成判定规则

| 判定维度 | 达成条件 | 偏差容忍度 |
|----------|----------|:---:|
| 转向 | 实际转角与目标转角偏差 ≤ 3° | 非铺装路面放宽至 5° |
| 动力 | 实际车速与目标车速偏差 ≤ 3 km/h | 紧急制动豁免 |
| 制动 | 实际制动压力与目标压力偏差 ≤ 0.5 MPa | 紧急制动放宽至 1.0 MPa |
| 姿态 | 横摆角速度与期望值偏差 ≤ 3°/s | — |
| 灯光 | 实际灯光状态与指令一致 | — |
| 档位 | 实际档位与目标档位一致 | — |

综合目标达成判定：所有参与模块的偏差均在容忍度内，且响应延迟最大值 ≤ 200ms，则判定为“完全达成”；部分模块未达标但非关键模块，判定为“部分达成”；关键模块（转向/制动）未达标，判定为“未达成”。


## 核心处理逻辑

```
FUNCTION closed_loop_feedback_main_loop():
    STATE_IDLE = IDLE
    STATE_COLLECT = COLLECTING
    STATE_JUDGE = JUDGING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化反馈收集表（指令ID → {模块ID: 偏差数据}）
    初始化超时计时器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：收到动作指令通知
        IF 收到 ad-mcc-01 动作指令通知:
            SET internal_state = STATE_COLLECT
            指令ID = 通知.指令ID
            根据指令类型确定需要收集反馈的模块列表
            初始化反馈收集表[指令ID] = {}
            超时计时器 = NOW()
            CONTINUE

        // 第3步：收集各模块偏差数据
        IF internal_state == STATE_COLLECT:
            指令ID = 当前活跃指令ID

            IF 收到转向偏差数据 AND 指令ID匹配:
                反馈收集表[指令ID]["转向"] = 偏差数据
            IF 收到动力偏差数据 AND 指令ID匹配:
                反馈收集表[指令ID]["动力"] = 偏差数据
            IF 收到制动偏差数据 AND 指令ID匹配:
                反馈收集表[指令ID]["制动"] = 偏差数据
            IF 收到姿态执行状态 AND 指令ID匹配:
                反馈收集表[指令ID]["姿态"] = 偏差数据
            IF 收到灯光执行状态 AND 指令ID匹配:
                反馈收集表[指令ID]["灯光"] = 偏差数据
            IF 收到档位执行状态 AND 指令ID匹配:
                反馈收集表[指令ID]["档位"] = 偏差数据

            // 检查是否所有需要的模块都已反馈
            需要模块列表 = 根据指令类型确定的模块列表
            已反馈模块 = 反馈收集表[指令ID].keys()

            IF 已反馈模块 包含所有 需要模块列表:
                SET internal_state = STATE_JUDGE
            ELSE IF NOW() - 超时计时器 > 500ms:
                // 超时，记录未反馈模块
                超时模块列表 = 需要模块列表 - 已反馈模块
                向 ad-mcc-01 发送闭环超时告警(指令ID, 超时模块列表)
                // 继续用已有数据进行判定
                SET internal_state = STATE_JUDGE

        // 第4步：执行闭环判定
        IF internal_state == STATE_JUDGE:
            偏差汇总 = 汇总各维度偏差(反馈收集表[指令ID])
            目标达成 = 判定目标达成(偏差汇总, 指令类型)
            综合偏差评级 = 计算综合评级(偏差汇总)
            响应延迟最大值 = MAX(各模块响应延迟)
            在线率 = 在线模块数 / 需要模块总数

            闭环回执 = 构建动作闭环回执(指令ID, 目标达成, 偏差汇总, 综合偏差评级, 响应延迟最大值, 在线率)
            偏差数据包 = 构建偏差数据汇总包(指令ID, 偏差汇总)

            向 ad-mcc-01 发送动作闭环回执
            向 ad-mcc-37 发送偏差数据汇总包
            向 ad-mcc-38 发送闭环事件记录

            清除反馈收集表[指令ID]
            SET internal_state = STATE_IDLE

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 部分模块反馈超时（>500ms） | 使用已收到的反馈进行判定，标记超时模块，上报告警 | 下一指令周期正常 |
| 关键模块（转向/制动）未反馈 | 判定为“未达成”，标记严重异常，上报告警 | 模块恢复反馈 |
| 指令ID不匹配（收到旧指令的延迟反馈） | 丢弃不匹配的反馈数据，记录日志 | — |
| 动作指令通知超时未收到任何反馈 | 500ms后触发超时告警，判定为“未达成” | 下一指令正常 |
| 紧急熔断 | 暂停收集，清空反馈表，保持最后闭环回执 | 紧急解除后恢复 |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 动作指令通知 | 只读 | ad-mcc-01 下发 |
| 内部调度总线 | 读 | 转向/动力/制动/姿态/灯光/档位偏差数据 | 只读 | 各偏差监控模块发送 |

| 内部调度总线 | 写 | 偏差数据汇总包 | 专属写入 | 向 ad-mcc-37 发送 |
| 内部调度总线 | 写 | 闭环事件记录 | 事件触发写入 | 向 ad-mcc-38 发送 |
| 内部调度总线 | 写 | 闭环超时告警 | 事件触发写入 | 向 ad-mcc-01 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 关键模块（转向、制动）反馈缺失或偏差严重超标时，必须判定为“未达成”，不得虚报达成 |
| S-02 | 闭环回执中的数据必须与各模块上报的原始偏差数据一致，不得修饰或篡改 |
| S-03 | 超时未反馈的模块必须明确标记，并在闭环回执中注明 |
| S-04 | 本模块仅负责执行结果的汇总与判定，不参与任何操控指令的生成或修改 |
