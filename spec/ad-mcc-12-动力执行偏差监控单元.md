# ad-mcc-12-动力执行偏差监控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-12 |
| 模块名称 | 动力执行偏差监控单元 |
| 所属分区 | 三、动力控制集群 |
| 核心职责 | 实时对比 ad-mcc-11 输出的最终目标车速/油门指令所对应的期望车速与车辆实际车速（来自 CAN 总线车速传感器），计算速度偏差与响应延迟。当速度偏差超过预设阈值时，触发动力异常告警并上报 ECC，同时根据偏差量级建议降级或检查动力系统。不参与任何操控指令的修改，仅做监控与告警 |
| 依赖模块 | ad-mcc-11（加速平顺滤波单元，提供最终油门目标指令及期望加速度，可据此推算出目标车速曲线）、车辆速度传感器（通过 CAN 总线提供实际车速） |
| 被依赖模块 | ad-mcc-01（小脑总控调度核心，接收速度偏差告警）、ad-mcc-03（全身运动状态归集中心，接收动力执行状态）、ECC-12（通过 CerebellumBus 接收严重偏差告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 速度偏差在容许范围内，动力系统正常响应 | 系统初始化完成，车速传感器在线 |
| 偏差预警 | `DEVIATION_WARNING` | 速度偏差超过预警阈值，动力响应偏慢或偏快 | 偏差持续 ≥ 3km/h 且 < 5km/h，持续 500ms |
| 偏差严重 | `DEVIATION_CRITICAL` | 速度偏差超过严重阈值，可能影响行车安全或动力系统故障 | 偏差 ≥ 5km/h 或响应延迟 > 200ms |
| 传感器离线 | `SENSOR_OFFLINE` | 车速传感器无信号或数据异常 | 传感器心跳超时 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 最终油门目标指令 | Struct（时间戳 + 滤波后油门开度_pct + 期望加速度_ms2 + 滤波方法 + 滤波强度） | ad-mcc-11 加速平顺滤波单元 | 每次滤波输出更新时 | **高** |
| 实际车速 | Float（km/h） | CAN 总线（车速传感器） | 实时，100Hz | **高** |
| 目标车速（如有） | Float（km/h） | ad-mcc-01（从 ECC 行驶意图中提取，或由 ad-mcc-11 结合期望加速度积分推算） | 每次 ECC 下发巡航/加速意图时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 动力执行状态 | Struct（目标车速 + 实际车速 + 速度偏差_kmh + 响应延迟_ms + 油门开度 + 在线状态） | ad-mcc-03 全身运动状态归集中心 | 每周期更新（100Hz） | **高** |
| 动力偏差告警 | Struct（偏差量 + 告警等级 + 建议动作 + 时间戳） | ad-mcc-01（通过内部调度总线）、ECC-12（通过 CerebellumBus，严重时） | 偏差超限时 | **高** |
| 传感器离线告警 | Struct（传感器ID + 最后有效时间 + 影响） | ad-mcc-01、ECC-12 | 车速传感器连续无信号 | **高** |


## 偏差判定阈值

| 偏差程度 | 速度偏差阈值 | 响应延迟阈值 | 告警等级 | 建议动作 |
|----------|:---:|:---:|:---:|----------|
| 正常 | < 2 km/h | < 100ms | 无 | 无需处理 |
| 轻微偏差 | 2 – 3 km/h | 100–150ms | 提示 | 记录日志，持续监控 |
| 预警 | 3 – 5 km/h | 150–200ms | 预警 | 上报 ECC，建议降低动力需求或检查动力系统 |
| 严重 | ≥ 5 km/h | > 200ms | 严重 | 触发降级，建议限制功率或紧急停车 |


## 核心处理逻辑

```
FUNCTION powertrain_deviation_monitor_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_WARNING = DEVIATION_WARNING
    STATE_CRITICAL = DEVIATION_CRITICAL
    STATE_OFFLINE = SENSOR_OFFLINE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化最后有效车速 = 0
    初始化传感器超时计时器 = 0
    初始化偏差持续计数器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE

        // 第2步：接收目标指令与实际车速
        IF 收到 ad-mcc-11 最终油门目标指令:
            目标油门 = 指令.滤波后油门开度_pct
            期望加速度 = 指令.期望加速度_ms2
            指令时间戳 = 指令.时间戳
            目标车速 = 获取目标车速()  // 来自ECC意图或推算

            // 第3步：获取实际车速
            实际车速 = 获取实际车速()
            车速有效 = 车速传感器在线且数据有效

            IF NOT 车速有效:
                传感器超时计时器 += 控制周期
                IF 传感器超时计时器 > 100ms:
                    SET internal_state = STATE_OFFLINE
                    使用最后有效车速作为实际车速
                    向 ad-mcc-01 发送传感器离线告警
                ELSE:
                    使用最后有效车速
            ELSE:
                传感器超时计时器 = 0
                IF internal_state == STATE_OFFLINE:
                    SET internal_state = STATE_NORMAL
                最后有效车速 = 实际车速

            // 第4步：计算偏差与延迟
            速度偏差 = 目标车速 - 实际车速
            响应延迟 = NOW() - 指令时间戳

            // 第5步：偏差判定（考虑持续时长）
            IF ABS(速度偏差) >= 5.0 OR 响应延迟 > 200ms:
                SET internal_state = STATE_CRITICAL
                告警等级 = "严重"
                建议动作 = "触发降级，检查动力系统"
            ELSE IF ABS(速度偏差) >= 3.0 OR 响应延迟 > 150ms:
                偏差持续计数器++
                IF 偏差持续计数器 > 50（500ms/10ms）:
                    SET internal_state = STATE_WARNING
                    告警等级 = "预警"
                    建议动作 = "降低动力需求，检查动力系统"
            ELSE:
                偏差持续计数器 = 0
                IF internal_state != STATE_OFFLINE:
                    SET internal_state = STATE_NORMAL
                告警等级 = "正常"
                建议动作 = ""

            // 第6步：生成动力执行状态
            动力执行状态 = {
                目标车速: 目标车速,
                实际车速: 实际车速,
                速度偏差_kmh: 速度偏差,
                油门开度: 目标油门,
                响应延迟_ms: 响应延迟,
                在线状态: 车速有效
            }
            向 ad-mcc-03 发送动力执行状态

            // 第7步：偏差超限时发送告警
            IF internal_state IN [STATE_WARNING, STATE_CRITICAL]:
                动力偏差告警 = {
                    偏差量: 速度偏差,
                    告警等级: 告警等级,
                    建议动作: 建议动作,
                    时间戳: NOW()
                }
                向 ad-mcc-01 发送动力偏差告警
                IF internal_state == STATE_CRITICAL:
                    向 ECC-12 发送动力偏差告警（通过 CerebellumBus）

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 车速传感器信号丢失（>100ms） | 标记传感器离线，使用最后有效车速，上报告警 | 传感器恢复信号 |
| 目标车速/油门指令超时未更新（>300ms） | 维持上一帧偏差评估，标记“指令可能过时”，偏差持续累积不触发新告警 | 指令恢复更新 |
| 速度偏差瞬间跳变（单帧偏差>10km/h） | 视为传感器噪声，暂时保持上一帧偏差等级，连续3帧确认后再升级 | 连续3帧偏差回落 |
| 紧急制动期间期望车速为0 | 仅对比实际车速是否为0，偏差容忍度临时扩大至5km/h | 紧急制动解除 |
| 紧急熔断 | 暂停监控，保留最后有效状态 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 最终油门目标指令 | 只读 | ad-mcc-11 发送 |
| 内部调度总线 | 读 | 目标车速（如有） | 只读 | ad-mcc-01 转发 |
| CAN 总线 | 读 | 实际车速 | 只读 | 实时 |
| 内部调度总线 | 写 | 动力执行状态 | 专属写入 | 向 ad-mcc-03 发送 |
| 内部调度总线 | 写 | 动力偏差告警 | 事件触发写入 | 向 ad-mcc-01 发送 |
| CerebellumBus | 写 | 动力偏差告警（严重） | 事件触发写入 | 向 ECC-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块仅做监控与告警，不参与任何操控指令的修改与生成 |
| S-02 | 车速传感器离线时必须明确标记并使用最后有效值填充，不可伪造在线状态 |
| S-03 | 严重速度偏差告警（≥5km/h）必须立即上报 ECC-12，触发安全降级评估 |
| S-04 | 偏差数据须与原始目标指令和传感器数据保持一致，不得篡改 |
