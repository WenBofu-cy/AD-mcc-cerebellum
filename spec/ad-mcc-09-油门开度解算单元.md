# ad-mcc-09-油门开度解算单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-09 |
| 模块名称 | 油门开度解算单元 |
| 所属分区 | 三、动力控制集群 |
| 核心职责 | 将 ECC 大脑通过 ad-mcc-01 下发的目标速度/加速度意图转化为具体的油门开度指令（0–100%）。基于车辆动力参数（最大功率、扭矩曲线、当前档位）、当前车速、路面坡度与摩擦系数，运用车辆纵向动力学模型，计算每一帧所需的油门踏板开度与期望加速度曲线。是连接 ECC 决策层与车辆底层驱动执行层之间的核心翻译模块。不参与任何场景判断与驾驶决策 |
| 依赖模块 | ad-mcc-01（小脑总控调度核心，下发动力调度指令）、ad-mcc-34（动力与制动参数管理单元，提供最大功率/扭矩/加速特性曲线）、ad-mcc-35（能源参数管理单元，提供电池/燃油当前状态）、ad-mcc-02（运动生理边界闸门，校验输出指令） |
| 被依赖模块 | ad-mcc-10（纵向冲击度约束单元，接收原始油门开度序列进行冲击度约束） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无动力调度指令，油门维持当前开度或回零 | 系统初始化完成，无动力调度指令 |
| 正常解算中 | `CALCULATING` | 正在基于车辆动力学模型将目标速度/加速度转化为油门开度 | 收到 ad-mcc-01 下发的动力调度指令 |
| 降级估算 | `DEGRADED_ESTIMATION` | 车辆动力参数部分缺失，使用保守默认值进行估算 | 最大功率/扭矩曲线等参数不可用 |
| 强制怠速 | `FORCED_IDLE` | 收到紧急制动或碰撞后响应指令，油门强制归零 | 收到紧急制动/碰撞后响应指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 纵向动力学解算模型

### 一、基础行驶阻力模型

```
F_total = F_roll + F_air + F_grade + F_accel

其中:
  F_roll  = m × g × Cr × cos(θ)     // 滚动阻力
  F_air   = 0.5 × ρ × Cd × A × v²   // 空气阻力
  F_grade = m × g × sin(θ)           // 坡度阻力
  F_accel = m × a_target             // 加速力

  m   = 车辆质量 (kg)
  g   = 重力加速度 (9.81 m/s²)
  Cr  = 滚动阻力系数（铺装路面≈0.015，非铺装≈0.04）
  θ   = 路面坡度角 (rad)
  ρ   = 空气密度 (1.225 kg/m³)
  Cd  = 风阻系数
  A   = 迎风面积 (m²)
  v   = 当前车速 (m/s)
  a_target = 目标加速度 (m/s²)
```

### 二、油门开度映射

```
// 计算需求功率
P_req = F_total × v / η_transmission

// 查询发动机/电机的功率-油门特性曲线
throttle_pct = PowerToThrottleMap(P_req, P_max, current_rpm)

其中:
  η_transmission = 传动系统效率 (0.85–0.95)
  P_max = 当前转速下的最大可用功率 (kW)
  PowerToThrottleMap = 查表函数，将需求功率映射为油门开度百分比
```

### 三、各驾驶模式油门响应特性

| 驾驶模式 | 油门响应延迟 | 最大油门开度 | 加速度上限 | 踏板曲线 |
|----------|:---:|:---:|:---:|:---:|
| 正常 | 标准 | 100% | 3.0 m/s² | 线性 |
| 一级降级 | 标准 | 80% | 2.5 m/s² | 线性 |
| 二级降级 | 标准 | 60% | 2.0 m/s² | 线性 |
| 三级降级 | — | 0%（强制怠速） | 0 | — |
| 非铺装道路 | 柔和 | 70% | 1.8 m/s² | 渐进曲线 |


## 核心处理逻辑

```
FUNCTION throttle_calculation_main_loop():
    STATE_IDLE = IDLE
    STATE_CALC = CALCULATING
    STATE_DEGRADED = DEGRADED_ESTIMATION
    STATE_IDLE_FORCED = FORCED_IDLE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化车辆参数（从 ad-mcc-34/35 加载）
    车辆质量 = 车辆参数.整备质量 + 载重估计
    最大功率 = 车辆参数.最大功率_kw
    风阻系数 = 车辆参数.风阻系数
    迎风面积 = 车辆参数.迎风面积_m2

    WHILE 系统运行中:
        // 第1步：最高优先级——紧急制动强制怠速
        IF 收到紧急制动/碰撞后响应指令:
            SET internal_state = STATE_IDLE_FORCED
            油门目标开度 = 0.0
            向 ad-mcc-10 发送油门目标开度序列(0.0, 原因="紧急制动强制怠速")
            CONTINUE

        // 第2步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第3步：车辆参数校验
        IF 车辆质量 <= 0 OR 最大功率 <= 0:
            SET internal_state = STATE_DEGRADED
            使用保守默认值（质量=1800kg, 功率=100kW）
            上报告警
        ELSE IF internal_state == STATE_DEGRADED:
            SET internal_state = STATE_IDLE

        // 第4步：接收动力调度指令
        IF 收到 ad-mcc-01 动力调度指令 AND internal_state NOT IN [STATE_IDLE_FORCED, STATE_PAUSED]:
            SET internal_state = STATE_CALC
            指令 = 动力调度指令
            当前车速 = 获取当前车速()
            当前油门 = 获取当前油门开度()
            当前坡度 = 获取当前路面坡度()
            摩擦系数 = 获取当前路面摩擦系数()

            // 4a. 目标值校验
            target_speed = 指令.目标速度_kmh
            target_accel = 指令.目标加速度_ms2

            // 降级模式油门开度上限
            IF 指令.模式标记 == "一级降级":
                最大油门开度 = 80
            ELSE IF 指令.模式标记 == "二级降级":
                最大油门开度 = 60
            ELSE:
                最大油门开度 = 100

            // 4b. 计算需求牵引力
            当前车速_ms = 当前车速 / 3.6
            需求牵引力 = 计算行驶阻力(当前车速_ms, 当前坡度, 摩擦系数) + 车辆质量 × target_accel

            // 4c. 计算需求功率
            需求功率_kw = 需求牵引力 × 当前车速_ms / 1000 / 传动效率

            // 4d. 功率限制（能源状态）
            最大可用功率 = MIN(最大功率, 获取当前能源允许的最大输出功率())

            // 4e. 油门开度映射
            IF 需求功率_kw <= 0:
                油门目标开度 = 0.0  // 减速场景，依赖制动系统
            ELSE:
                功率比 = MIN(需求功率_kw / 最大可用功率, 1.0)
                // 查表或线性映射
                油门目标开度 = 功率比 × 最大油门开度

            // 4f. 非铺装模式油门响应柔化
            IF 指令.模式标记 == "非铺装道路":
                油门变化量 = 油门目标开度 - 当前油门
                最大变化量 = 5.0  // 每帧最多变化 5%
                IF ABS(油门变化量) > 最大变化量:
                    油门目标开度 = 当前油门 + SIGN(油门变化量) × 最大变化量

            // 4g. 计算期望加速度
            期望加速度 = (需求牵引力 - 计算行驶阻力(当前车速_ms, 当前坡度, 摩擦系数)) / 车辆质量

            // 4h. 生成油门目标开度序列
            油门序列 = {
                时间戳: NOW(),
                目标油门开度_pct: 油门目标开度,
                期望加速度_ms2: 期望加速度,
                解算方法: "纵向动力学模型",
                置信度: IF internal_state == STATE_DEGRADED THEN 0.7 ELSE 0.95
            }

            // 4i. 输出解算结果
            向 ad-mcc-10 发送油门目标开度序列
            向 ad-mcc-01 上报解算状态

            SET internal_state = STATE_IDLE

        SLEEP 10ms


FUNCTION 计算行驶阻力(车速_ms, 坡度_rad, 摩擦系数):
    滚动阻力系数 = IF 摩擦系数 < 0.4 THEN 0.04 ELSE 0.015
    F_roll = 车辆质量 × 9.81 × 滚动阻力系数 × cos(坡度_rad)
    F_air = 0.5 × 1.225 × 风阻系数 × 迎风面积 × 车速_ms²
    F_grade = 车辆质量 × 9.81 × sin(坡度_rad)
    RETURN F_roll + F_air + F_grade
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 车辆动力参数不可用 | 使用保守默认值（质量=1800kg, 功率=100kW），标记为降级估算 | 参数恢复 |
| 能源系统限制输出功率（如电池过热/亏电） | 油门开度上限受能源允许的最大输出功率限制 | 能源状态恢复 |
| 目标加速度超出车辆动力极限 | 自动截断至当前车速下最大可行加速度，记录修正日志 | — |
| 下坡路段目标速度为负（减速） | 油门开度设为0，依赖制动系统或能量回收 | — |
| 紧急制动指令 | 油门强制归零，状态切换至 FORCED_IDLE | 紧急制动解除 |
| 车速信号丢失 | 假设车速=当前路段限速×0.7（保守中速），标记数据质量降级 | 车速信号恢复 |
| 紧急熔断 | 暂停解算，维持当前油门开度不变 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 紧急制动或碰撞后响应指令为最高优先级，收到后油门必须立即强制归零（FORCED_IDLE） |
| S-02 | 解算出的油门开度必须严格约束在当前驾驶模式允许的最大开度以内 |
| S-03 | 车辆动力参数缺失时，必须使用保守默认值并明确标记“降级估算” |
| S-04 | 本模块仅输出目标油门开度序列，不直接操控驱动电机或节气门 |
| S-05 | 下坡或减速场景下，油门开度必须归零，不得通过油门补偿来对抗制动系统 |
