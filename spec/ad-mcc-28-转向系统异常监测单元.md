# ad-mcc-28-转向系统异常监测单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-28 |
| 模块名称 | 转向系统异常监测单元 |
| 所属分区 | 八、硬件异常应急防护 |
| 核心职责 | 实时监测转向电机电流、转角传感器双路信号一致性及转向系统通信状态，判断转向系统是否存在电气故障、机械卡滞或传感器失效。当检测到异常时，根据故障严重等级触发分级降级策略：轻微异常时发出预警并限制转向速率；严重异常时触发二级降级，限制车速至40km/h并请求人工接管；致命异常时触发三级降级并禁止自动驾驶。不参与转向控制决策，仅做故障诊断与降级触发 |
| 依赖模块 | 转向电机电流传感器（CAN 总线）、方向盘转角传感器（CAN 总线，双路冗余）、ad-mcc-01（小脑总控调度核心，接收降级触发请求）、ad-mcc-02（运动生理边界闸门，校验降级后的转向参数限制） |
| 被依赖模块 | ad-mcc-01（接收异常告警与降级触发）、ad-mcc-38（执行日志记录单元，记录故障事件）、ECC-12（通过 CerebellumBus 接收严重故障告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监测 | `NORMAL_MONITOR` | 转向系统运行正常，各项指标在容许范围内 | 系统初始化完成，传感器在线且数据有效 |
| 轻微异常预警 | `MINOR_FAULT` | 检测到转向电机电流偏高或转角传感器偏差接近阈值，暂时不影响控制 | 电机电流 > 额定值110% 或 双路偏差 > 3°，持续 500ms |
| 严重异常降级 | `SEVERE_FAULT` | 转向电机过载或传感器偏差过大，可能影响转向控制精度 | 电机电流 > 额定值120% 或 双路偏差 > 5° 或 通信错误率 > 1% |
| 致命异常禁驾 | `CRITICAL_FAULT` | 转向系统完全失效或传感器双路均故障，禁止自动驾驶 | 电机堵转/短路 或 转角传感器双路同时失效 或 通信总线故障 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 转向电机电流 | Float（A） | CAN 总线（电机电流传感器） | 实时，100Hz | **高** |
| 转角传感器数据（主路） | Float（°） | CAN 总线（转角传感器主路） | 实时，100Hz | **高** |
| 转角传感器数据（冗余路） | Float（°） | CAN 总线（转角传感器冗余路） | 实时，100Hz | **高** |
| 转向系统故障码 | Struct（故障码 + 故障等级 + 描述） | CAN 总线（EPS 控制器） | 事件触发 | **高** |
| 转向通信总线状态 | Struct（CAN 帧错误率 + 节点心跳） | ad-mcc-31 通信总线异常监测单元 | 周期性，10Hz | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 转向系统异常告警 | Struct（故障类型 + 严重等级 + 当前值 + 阈值 + 建议动作） | ad-mcc-01（内部调度总线）、ECC-12（严重时通过 CerebellumBus） | 故障触发时 | **高** |
| 降级触发请求 | Struct（目标降级等级 + 触发原因 + 限速值 + 转向限制参数） | ad-mcc-01（小脑总控调度核心） | 严重/致命异常时 | **高** |
| 转向健康状态上报 | Struct（电机电流 + 传感器偏差 + 通信状态 + 当前故障等级） | ad-mcc-01、ad-mcc-38 | 周期性，每1秒 | 普通 |
| 故障事件记录 | Struct（故障类型 + 发生时间 + 当前值 + 持续时长） | ad-mcc-38 执行日志记录单元 | 故障发生与恢复时 | 普通 |


## 故障判定阈值

| 故障类型 | 正常范围 | 预警阈值 | 严重阈值 | 致命阈值 | 判定延时 |
|----------|:---:|:---:|:---:|:---:|:---:|
| 转向电机电流 | ≤ 额定值 | > 额定值110% | > 额定值120% | 堵转/短路 | 500ms 持续 |
| 转角传感器双路偏差 | ≤ 2° | > 3° | > 5° | 双路均失效 | 500ms 持续 |
| 通信帧错误率 | ≤ 0.5% | > 0.5% | > 1% | 总线完全中断 | 1s 持续 |
| 传感器心跳超时 | ≤ 50ms | > 50ms | > 100ms | > 200ms | 立即/超时判定 |


## 核心处理逻辑

```
FUNCTION steering_fault_monitor_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_MINOR = MINOR_FAULT
    STATE_SEVERE = SEVERE_FAULT
    STATE_CRITICAL = CRITICAL_FAULT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化故障持续计数器（电流、偏差、通信）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL
            重置所有故障计数器

        // 第2步：获取传感器数据
        电机电流 = 获取转向电机电流()
        主路转角 = 获取主路转角传感器值()
        冗余转角 = 获取冗余转角传感器值()
        故障码 = 获取EPS故障码()
        通信状态 = 获取转向通信总线状态()

        // 第3步：传感器健康判定
        传感器故障 = False
        IF 主路转角超时 > 200ms OR 冗余转角超时 > 200ms:
            传感器故障 = True
            IF 主路超时 AND 冗余超时:
                双路失效 = True
            ELSE:
                双路失效 = False

        IF 双路失效:
            SET internal_state = STATE_CRITICAL
            向 ad-mcc-01 发送降级触发请求(三级降级, "转角传感器双路失效")
            向 ECC-12 发送转向系统异常告警("致命", "传感器双路失效")
            CONTINUE

        IF 传感器故障 AND NOT 双路失效:
            使用有效的一路作为实际值
            标记传感器部分故障

        // 第4步：电流判定
        电流状态 = "正常"
        IF 电机电流 > 额定值 × 1.2 OR 故障码包含"过载":
            电流状态 = "严重"
            电流超限计数器++
        ELSE IF 电机电流 > 额定值 × 1.1:
            电流状态 = "预警"
            电流超限计数器++
        ELSE:
            电流超限计数器 = 0

        // 第5步：传感器偏差判定
        偏差状态 = "正常"
        IF NOT 双路失效:
            转角偏差 = ABS(主路转角 - 冗余转角)
            IF 转角偏差 > 5.0:
                偏差状态 = "严重"
                偏差超限计数器++
            ELSE IF 转角偏差 > 3.0:
                偏差状态 = "预警"
                偏差超限计数器++
            ELSE:
                偏差超限计数器 = 0

        // 第6步：通信状态判定
        通信状态_flag = "正常"
        IF 通信状态.帧错误率 > 1.0:
            通信状态_flag = "严重"
        ELSE IF 通信状态.帧错误率 > 0.5:
            通信状态_flag = "预警"

        // 第7步：综合故障等级判定
        IF 电流状态 == "严重" OR 偏差状态 == "严重" OR 通信状态_flag == "严重" OR 故障码.等级 >= 严重:
            IF 电流超限计数器 > 50（=500ms）OR 偏差超限计数器 > 50:
                SET internal_state = STATE_SEVERE
                向 ad-mcc-01 发送降级触发请求(二级降级, 限速40km/h, 转向速率限制150°/s)
                向 ECC-12 发送转向系统异常告警("严重", 故障原因)
        ELSE IF 电流状态 == "预警" OR 偏差状态 == "预警" OR 通信状态_flag == "预警":
            IF 电流超限计数器 > 50 OR 偏差超限计数器 > 50:
                SET internal_state = STATE_MINOR
                向 ad-mcc-01 发送转向系统异常告警("预警", 故障原因)
        ELSE:
            IF internal_state != STATE_NORMAL:
                记录故障恢复日志
                SET internal_state = STATE_NORMAL
            重置所有故障计数器

        // 第8步：周期性状态上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 上报转向健康状态
            向 ad-mcc-38 记录状态快照

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 转角传感器单路失效 | 使用有效的一路继续工作，标记传感器降级，上报告警 | 失效路恢复 |
| 转角传感器双路失效 | 立即触发三级降级，禁止自动驾驶，转向控制交由人工 | 至少一路恢复 |
| 转向电机过载 | 限制转向速率至150°/s，触发二级降级 | 电流恢复正常且持续1秒 |
| 通信总线故障 | 切换至冗余总线，若无冗余则触发三级降级 | 总线恢复 |
| 故障恢复后 | 所有指标正常且持续3秒后，解除降级状态 | — |
| 紧急熔断 | 暂停监测，保持最后故障状态 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| CAN 总线 | 读 | 转向电机电流 | 只读 | 实时 |
| CAN 总线 | 读 | 转角传感器主路/冗余 | 只读 | 实时 |
| CAN 总线 | 读 | EPS 故障码 | 只读 | 事件触发 |
| 内部调度总线 | 读 | 转向通信总线状态 | 只读 | ad-mcc-31 提供 |
| 内部调度总线 | 写 | 转向系统异常告警 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 降级触发请求 | 事件触发写入 | 向 ad-mcc-01 发送 |
| CerebellumBus | 写 | 严重/致命异常告警 | 事件触发写入 | 向 ECC-12 发送 |
| 内部调度总线 | 写 | 转向健康状态上报 | 周期性写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 故障事件记录 | 事件触发写入 | 向 ad-mcc-38 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 转角传感器双路失效时，必须立即禁止自动驾驶，不得依赖历史数据或模型推算 |
| S-02 | 转向电机过载必须限制转向速率，防止电机损坏导致转向失控 |
| S-03 | 故障恢复后需持续监测 3 秒确认正常，方可解除降级，防止间歇性故障反复触发 |
| S-04 | 本模块仅做故障诊断与降级触发，不直接干预转向执行器控制 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M28-01 | `NORMAL_MONITOR`，电流正常 | 电机电流=额定值，偏差=1° | 保持正常，无告警 |
| TC-M28-02 | `NORMAL_MONITOR`，电流偏高持续 | 电机电流=额定值115%，持续600ms | 进入 `MINOR_FAULT`，发送预警告警 |
| TC-M28-03 | `NORMAL_MONITOR`，传感器偏差过大 | 双路偏差=6°，持续600ms | 进入 `SEVERE_FAULT`，触发二级降级 |
| TC-M28-04 | `NORMAL_MONITOR`，双路传感器失效 | 主路超时200ms，冗余超时200ms | 进入 `CRITICAL_FAULT`，触发三级降级 |
| TC-M28-05 | `SEVERE_FAULT`，故障恢复 | 电流正常，偏差正常，持续3秒 | 恢复 `NORMAL_MONITOR` |
| TC-M28-06 | `SYSTEM_PAUSED`，收到熔断 | 紧急熔断指令 | 暂停监测，保持最后状态 |
