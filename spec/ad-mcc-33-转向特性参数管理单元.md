# ad-mcc-33-转向特性参数管理单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-33 |
| 模块名称 | 转向特性参数管理单元 |
| 所属分区 | 九、多车型自适应适配 |
| 核心职责 | 存储并管理车辆出厂前一次性标定的转向系统特性参数（转向比、最大方向盘转角、转向机响应特性、转向助力曲线等），为 MCC 运动小脑各转向相关执行模块提供统一的转向特性参数查询服务。确保参数来源唯一、版本一致，避免各模块独立维护参数导致的不一致。不参与任何驾驶决策或控制，仅提供参数数据的存取与校验 |
| 依赖模块 | 车辆出厂标定系统（通过配置文件或诊断接口写入，系统初始化时加载） |
| 被依赖模块 | ad-mcc-04（方向盘转角解算单元，查询转向比、最大转角）、ad-mcc-05（转向平顺滤波单元，查询转向机响应特性）、ad-mcc-08（非铺装路面转向适配单元，查询最大转角限制）、ad-mcc-19（横摆稳定控制单元，查询转向比用于期望横摆计算） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL_SERVICE` | 所有转向特性参数已成功加载并通过校验，可正常响应查询 | 系统初始化完成，参数加载成功 |
| 参数缺失降级 | `DEGRADED_SERVICE` | 部分参数加载失败或未标定，使用保守默认值响应查询，置信度标记为降低 | 参数文件校验失败或部分字段缺失 |
| 参数更新中 | `UPDATING` | 正在进行参数更新（OTA 升级或诊断工具写入），暂不响应查询 | 收到参数更新请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 出厂标定参数文件 | Struct（转向传动比 + 方向盘最大转角 + 转向机响应时间_ms + 助力曲线类型 + 助力增益） | 出厂标定系统（配置文件/诊断接口） | 系统初始化加载，参数更新时 | **高** |
| 参数查询请求 | Struct（请求模块ID + 查询参数列表） | ad-mcc-04/05/08/19 等各转向相关模块 | 各模块初始化或按需查询 | 普通 |
| 参数更新指令 | Struct（更新参数列表 + 更新来源 + 校验和） | OTA 升级系统或诊断工具 | 车辆维护时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 转向特性参数查询回执 | Struct（请求参数名 + 参数值 + 单位 + 置信度 + 是否为默认值） | 各请求模块 | 收到查询请求后 | 普通 |
| 参数校验状态上报 | Struct（参数总数 + 已加载数 + 默认值替代数 + 校验状态） | ad-mcc-01（内部调度总线） | 系统初始化完成或参数更新后 | 普通 |
| 参数更新完成回执 | Struct（更新结果 + 校验状态 + 生效时间） | OTA/诊断系统 | 参数更新完成时 | 普通 |
| 参数异常告警 | Struct（缺失参数列表 + 默认值 + 影响模块） | ad-mcc-01、ad-mcc-38 | 参数缺失并使用默认值时 | 普通 |

## 核心处理逻辑

```
FUNCTION steering_params_manager_main_loop():
    STATE_NORMAL = NORMAL_SERVICE
    STATE_DEGRADED = DEGRADED_SERVICE
    STATE_UPDATING = UPDATING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_UPDATING
    初始化参数存储表（全部加载默认值）
    标记所有参数为“默认值”

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：系统初始化时加载参数
        IF internal_state == STATE_UPDATING:
            标定文件 = 加载出厂标定参数文件()
            IF 标定文件.校验通过:
                FOR EACH 参数 IN 标定文件:
                    IF 参数值在有效范围内:
                        更新参数存储表(参数名, 参数值)
                        标记参数为“已标定”
                    ELSE:
                        保持默认值
                        标记参数为“无效标定值”
                        记录告警日志
                SET internal_state = STATE_NORMAL
            ELSE:
                SET internal_state = STATE_DEGRADED
                记录告警日志("参数文件校验失败，全部使用默认值")
                CONTINUE

        // 第3步：参数更新指令
        IF 收到参数更新指令:
            SET internal_state = STATE_UPDATING
            校验更新数据完整性()
            IF 校验通过:
                备份当前参数()
                应用新参数()
                SET internal_state = STATE_NORMAL
                向更新发起方发送参数更新完成回执(成功)
            ELSE:
                恢复备份参数()
                SET internal_state = STATE_NORMAL
                向更新发起方发送参数更新完成回执(失败)

        // 第4步：响应参数查询请求
        IF 收到参数查询请求:
            FOR EACH 查询参数 IN 请求.查询参数列表:
                参数值 = 参数存储表[查询参数]
                是否为默认 = 参数标记表[查询参数] == "默认值"
                置信度 = IF 是否为默认 AND internal_state == STATE_DEGRADED THEN 0.7 ELSE 1.0

                向请求模块返回查询回执(查询参数, 参数值, 单位, 置信度, 是否为默认)

        // 第5步：周期性状态上报
        IF 初始化完成 OR 参数更新完成:
            统计参数加载情况(总数, 已标定数, 默认值替代数)
            向 ad-mcc-01 上报参数校验状态

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 出厂标定文件缺失或损坏 | 全部参数使用保守默认值，标记 `DEGRADED_SERVICE`，置信度 0.7 | 重新加载有效标定文件 |
| 单个参数值超出有效范围 | 该参数使用默认值，其他参数正常使用 | 参数值被修正后重新加载 |
| 参数更新过程中系统断电 | 上电后回滚至更新前备份参数 | 重新尝试更新 |
| 查询不存在的参数名 | 返回错误码“参数不存在”，记录日志 | — |
| 紧急熔断 | 暂停服务，保持当前参数不变 | 紧急解除后恢复 |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 出厂标定参数为只读数据，运行时不得被任何控制模块修改 |
| S-02 | 参数更新必须经过完整性校验与人工确认，更新前必须备份当前参数 |
| S-03 | 参数缺失时必须使用保守默认值，确保转向控制不因参数缺失而失效 |
| S-04 | 本模块仅提供参数查询服务，不参与任何车辆控制决策 |
