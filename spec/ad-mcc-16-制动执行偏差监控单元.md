# ad-mcc-16-制动执行偏差监控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-16 |
| 模块名称 | 制动执行偏差监控单元 |
| 所属分区 | 四、制动控制集群 |
| 核心职责 | 实时对比 ad-mcc-15 输出的最终制动压力目标指令与车辆实际制动主缸压力（来自 CAN 总线压力传感器），计算压力偏差与制动响应延迟。当偏差超过预设阈值时，触发制动异常告警并上报 ECC，同时根据偏差量级建议降级或紧急检查。同时将偏差数据周期性推送至 ad-mcc-36 执行闭环反馈单元，供运动闭环回执与运动质量评估使用。不参与任何操控指令的修改，仅做监控与告警 |
| 依赖模块 | ad-mcc-15（制动平顺防点头单元，提供最终平顺化制动压力指令）、制动压力传感器（通过 CAN 总线提供实际制动主缸压力）、ad-mcc-36（执行闭环反馈单元，消费偏差数据）、ad-mcc-38（执行日志记录单元，记录偏差事件） |
| 被依赖模块 | ad-mcc-01（小脑总控调度核心，接收制动偏差告警）、ad-mcc-03（全身运动状态归集中心，接收制动执行状态）、ECC-12（通过 CerebellumBus 接收严重偏差告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 制动压力偏差在容许范围内，制动系统正常响应 | 系统初始化完成，制动压力传感器在线 |
| 偏差预警 | `DEVIATION_WARNING` | 制动压力偏差超过预警阈值，制动建压偏慢或偏快 | 压力偏差 ≥ 0.5 MPa 且 < 1.0 MPa，持续 500ms |
| 偏差严重 | `DEVIATION_CRITICAL` | 制动压力偏差超过严重阈值，可能制动失效或管路泄漏 | 压力偏差 ≥ 1.0 MPa 或响应延迟 > 200ms |
| 传感器离线 | `SENSOR_OFFLINE` | 制动压力传感器无信号或数据异常 | 传感器心跳超时（>100ms） |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 平顺化制动压力指令 | Struct（时间戳 + 修正后压力_MPa + 原始压力_MPa + 防点头状态 + 修正幅度） | ad-mcc-15 制动平顺防点头单元 | 每周期（200Hz） | **高** |
| 实际制动主缸压力 | Float（MPa） | CAN 总线（制动压力传感器） | 实时，200Hz | **高** |
| 制动压力传感器健康状态 | Struct（传感器ID + 在线状态 + 数据校验通过 + 信号质量） | 传感器自诊断模块 | 周期性，1Hz | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 制动执行状态 | Struct（目标压力 + 实际压力 + 压力偏差_MPa + 响应延迟_ms + 在线状态） | ad-mcc-03 全身运动状态归集中心 | 每周期更新（200Hz） | **高** |
| 制动偏差超限告警 | Struct（告警类型 + 目标值 + 实际值 + 偏差量 + 持续时长 + 建议动作） | ad-mcc-01（小脑总控调度核心） | 偏差超阈值持续 > 500ms（严重则立即） | **高** |
| 制动执行偏差数据 | Struct（指令ID + 压力偏差_MPa + 速率偏差_pct + 响应延迟_ms + 在线状态） | ad-mcc-36（执行闭环反馈单元） | 每次指令执行周期结束时 | **高** |
| 传感器异常告警 | Struct（传感器ID + 异常类型 + 最后有效值 + 影响评估） | ad-mcc-01、ad-mcc-38 | 制动压力传感器离线或数据校验失败 | **高** |
| 偏差监控状态上报 | Struct（当前状态 + 最近偏差量 + 告警计数 + 传感器健康） | ad-mcc-01 | 周期性，每1秒 | 普通 |


## 偏差判定阈值

| 偏差程度 | 压力偏差阈值 | 响应延迟阈值 | 告警等级 | 判定延迟 | 建议动作 |
|----------|:---:|:---:|:---:|:---:|----------|
| 正常 | < 0.3 MPa | < 100ms | 无 | — | 无需处理 |
| 轻微偏差 | 0.3 – 0.5 MPa | 100–150ms | 提示 | — | 记录日志，持续监控 |
| 预警 | 0.5 – 1.0 MPa | 150–200ms | 预警 | 500ms 持续 | 上报 ECC，建议降低制动需求或检查管路 |
| 严重 | ≥ 1.0 MPa | > 200ms | 严重 | 立即 | 触发降级，检查制动系统，准备紧急停车 |


## 核心处理逻辑

```
FUNCTION brake_deviation_monitor_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_WARNING = DEVIATION_WARNING
    STATE_CRITICAL = DEVIATION_CRITICAL
    STATE_OFFLINE = SENSOR_OFFLINE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化最后有效压力 = 0.0
    初始化传感器超时计时器 = 0
    初始化告警抑制 = {pressure: False, latency: False}

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL
            重置告警抑制标记

        // 第2步：传感器健康检查
        传感器状态 = 获取制动压力传感器健康状态()
        IF 传感器状态.离线 OR NOT 传感器状态.数据校验通过:
            SET internal_state = STATE_OFFLINE
            使用最后有效压力作为实际压力
            标记数据质量 = "降级"
            IF 传感器超时计时器 == 0:
                向 ad-mcc-01 发送传感器异常告警
            传感器超时计时器 += 控制周期
        ELSE:
            传感器超时计时器 = 0
            IF internal_state == STATE_OFFLINE:
                SET internal_state = STATE_NORMAL

        // 第3步：接收目标压力与实际压力
        IF 收到 ad-mcc-15 平顺化制动压力指令:
            目标压力 = 指令.修正后压力_MPa
            指令时间戳 = 指令.时间戳
            实际压力 = IF 传感器在线 THEN 获取实际压力() ELSE 最后有效压力
            IF 传感器在线:
                最后有效压力 = 实际压力

            // 第4步：计算偏差与延迟
            压力偏差 = 实际压力 - 目标压力  // 正值=超调，负值=不足
            响应延迟 = NOW() - 指令时间戳

            // 第5步：噪声过滤——瞬间跳变检测
            IF ABS(压力偏差 - 上一帧偏差) > 2.0 MPa:
                噪声计数器++
                IF 噪声计数器 < 3:
                    压力偏差 = 上一帧偏差
                ELSE:
                    噪声计数器 = 0
            ELSE:
                噪声计数器 = 0
            上一帧偏差 = 压力偏差

            // 第6步：偏差判定
            告警触发 = False
            告警类型 = ""
            告警等级 = ""

            IF ABS(压力偏差) >= 1.0 OR 响应延迟 > 200ms:
                SET internal_state = STATE_CRITICAL
                告警触发 = True
                告警等级 = "严重"
                IF ABS(压力偏差) >= 1.0:
                    告警类型 += "严重压力偏差"
                IF 响应延迟 > 200ms:
                    告警类型 += IF 告警类型 != "" THEN " + 严重响应延迟" ELSE "严重响应延迟"
                建议动作 = "触发降级，检查制动系统"
            ELSE IF ABS(压力偏差) >= 0.5 OR 响应延迟 > 150ms:
                偏差超限计数器++
                IF 偏差超限计数器 > 100（=500ms/5ms）:
                    SET internal_state = STATE_WARNING
                    告警触发 = True
                    告警等级 = "预警"
                    IF ABS(压力偏差) >= 0.5:
                        告警类型 += "压力偏差超限"
                    IF 响应延迟 > 150ms:
                        告警类型 += IF 告警类型 != "" THEN " + 响应延迟超限" ELSE "响应延迟超限"
                    建议动作 = "降低制动需求，检查制动管路"
            ELSE:
                偏差超限计数器 = 0
                IF internal_state NOT IN [STATE_OFFLINE, STATE_PAUSED]:
                    IF internal_state != STATE_NORMAL:
                        记录恢复日志("制动偏差恢复")
                    SET internal_state = STATE_NORMAL
                    重置告警抑制标记

            // 第7步：生成制动执行状态
            制动执行状态 = {
                目标压力: 目标压力,
                实际压力: 实际压力,
                压力偏差_MPa: 压力偏差,
                响应延迟_ms: 响应延迟,
                在线状态: 传感器在线
            }
            向 ad-mcc-03 发送制动执行状态

            // 第8步：上报告警（含抑制机制）
            IF 告警触发:
                IF NOT 告警抑制已激活(告警类型):
                    SET 告警抑制标记(告警类型)
                    向 ad-mcc-01 发送制动偏差超限告警(
                        告警类型=告警类型,
                        目标值=目标压力,
                        实际值=实际压力,
                        偏差量=ABS(压力偏差),
                        持续时长=偏差超限计数器 × 5ms,
                        建议动作=建议动作
                    )
                    IF 告警等级 == "严重":
                        向 ECC-12 发送制动偏差超限告警（通过 CerebellumBus）

            // 第9步：推送偏差数据至闭环反馈单元
            偏差数据 = {
                指令ID: 指令.msg_id,
                压力偏差_MPa: 压力偏差,
                响应延迟_ms: 响应延迟,
                在线状态: IF internal_state == STATE_OFFLINE THEN "降级" ELSE "正常"
            }
            向 ad-mcc-36 发送制动执行偏差数据

            // 第10步：偏差事件记录
            IF ABS(压力偏差) > 0.3 OR 响应延迟 > 100ms:
                向 ad-mcc-38 发送偏差事件记录（目标压力、实际压力、偏差量、时间戳）

        // 第11步：周期性状态上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 发送偏差监控状态上报(
                当前状态=internal_state,
                最近偏差量=ABS(压力偏差),
                告警计数=告警总数,
                传感器健康=传感器状态摘要
            )

        SLEEP 5ms  // 200Hz
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 制动压力传感器信号丢失（>100ms） | 标记传感器离线，使用最后有效值，上报告警 | 传感器恢复信号 |
| 目标压力指令超时未更新（>200ms） | 维持上一帧偏差评估，标记“指令可能过时” | 指令恢复更新 |
| 压力偏差瞬间跳变（单帧偏差>2.0MPa） | 视为传感器噪声，连续3帧确认后采纳；未达3帧则沿用上帧偏差值 | 连续3帧偏差回落或持续确认真实跳变 |
| 同一告警反复触发 | 告警抑制标记防止重复上报，待偏差恢复后重置 | 偏差恢复至正常范围 |
| 紧急熔断 | 暂停监控，保留最后有效状态 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 平顺化制动压力指令 | 只读 | ad-mcc-15 发送 |
| CAN 总线 | 读 | 实际制动主缸压力 | 只读 | 实时，200Hz |
| 传感器自诊断总线 | 读 | 制动压力传感器健康状态 | 只读 | 周期性，1Hz |
| 内部调度总线 | 写 | 制动执行状态 | 专属写入 | 向 ad-mcc-03 发送 |
| 内部调度总线 | 写 | 制动偏差超限告警 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 制动执行偏差数据 | 周期性写入 | 向 ad-mcc-36 发送 |
| 内部调度总线 | 写 | 传感器异常告警 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 偏差监控状态上报 | 周期性写入 | 向 ad-mcc-01 发送 |
| CerebellumBus | 写 | 制动偏差超限告警（严重） | 事件触发写入 | 向 ECC-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块仅做监控与告警，不参与任何操控指令的修改与生成 |
| S-02 | 制动压力传感器离线时必须明确标记并使用最后有效值填充，不可伪造在线状态 |
| S-03 | 严重制动压力偏差告警（≥1.0 MPa）必须立即上报 ECC-12，触发安全降级评估，不得等待判定延迟 |
| S-04 | 偏差数据须与原始目标指令和传感器数据保持一致，不得篡改 |
| S-05 | 告警抑制机制不得遗漏新的告警类型。同一类型告警可抑制，不同类型告警须分别上报 |
