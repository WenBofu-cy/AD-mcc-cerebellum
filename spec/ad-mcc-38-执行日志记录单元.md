# ad-mcc-38-执行日志记录单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-38 |
| 模块名称 | 执行日志记录单元 |
| 所属分区 | 十、执行反馈与日志 |
| 核心职责 | 全链路记录 MCC 运动小脑所有操控指令、实际执行结果、偏差数据、异常事件及故障信息，生成不可篡改的审计日志。日志存储周期 ≥ 6 个月，支持按时间、模块、事件类型等多维度检索。为事故追溯、责任判定、系统调试及离线复盘提供完整的数据基础。不参与任何驾驶决策，仅负责日志的记录、存储与检索服务 |
| 依赖模块 | ad-mcc-01 至 ad-mcc-37（所有模块均可向本模块推送日志事件）、ad-mcc-36（执行闭环反馈单元，推送闭环事件记录）、ad-mcc-37（运动质量评估单元，推送质量趋势记录） |
| 被依赖模块 | ad-mcc-01（小脑总控调度核心，查询历史日志供决策追溯）、ECC-12（通过 CerebellumBus 查询事故相关日志）、离线复盘系统（通过诊断接口导出日志） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常记录 | `NORMAL_LOGGING` | 正常接收并存储各模块推送的日志事件 | 系统初始化完成，存储介质就绪 |
| 存储空间不足 | `LOW_STORAGE` | 日志存储空间低于警戒阈值（< 20%），触发旧日志滚动清理 | 剩余存储空间 < 20% |
| 存储故障 | `STORAGE_FAULT` | 存储介质写入失败或不可用，日志仅暂存内存缓冲区 | 存储写入失败连续3次 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 日志存储策略

### 一、日志分类与优先级

| 日志类别 | 优先级 | 最短保留期 | 不可覆写标记 | 说明 |
|----------|:---:|:---:|:---:|------|
| 操控指令日志 | 高 | 6 个月 | 否 | 每次指令下发记录 |
| 执行结果日志 | 高 | 6 个月 | 否 | 与指令日志关联 |
| 偏差事件日志 | 高 | 1 年 | 是（严重偏差） | 严重偏差不可覆写 |
| 异常/故障日志 | 紧急 | 3 年 | 是 | 所有故障日志不可覆写 |
| 质量评估日志 | 普通 | 6 个月 | 否 | — |
| 状态变更日志 | 普通 | 6 个月 | 否 | — |

### 二、存储滚动策略

当存储空间低于 20% 时：
1. 优先删除最旧的、可覆写的普通优先级日志。
2. 若删除后仍不足，删除高优先级可覆写日志（但保留至少 6 个月）。
3. 禁止删除标记为“不可覆写”的日志。
4. 若不可覆写日志占比超过 60%，发出存储升级建议告警。


## 核心处理逻辑

```
FUNCTION execution_logger_main_loop():
    STATE_NORMAL = NORMAL_LOGGING
    STATE_LOW = LOW_STORAGE
    STATE_FAULT = STORAGE_FAULT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化日志存储（文件系统或数据库）
    初始化内存缓冲区（用于存储故障时的暂存）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            强制将缓冲区内容写入存储（若可用）
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收并分类存储日志事件
        日志事件 = 获取下一个待记录事件()
        WHILE 日志事件 != None:
            根据日志事件.事件类型确定日志类别
            添加时间戳、序列号、来源模块

            IF internal_state == STORAGE_FAULT:
                暂存至内存缓冲区
                IF 缓冲区使用率 > 80%:
                    丢弃最旧的可覆写缓冲区条目
            ELSE:
                写入持久化存储
                IF 写入失败:
                    写入失败计数器++
                    IF 写入失败计数器 >= 3:
                        SET internal_state = STATE_FAULT
                        向 ad-mcc-01 发送存储故障告警
                        暂存至缓冲区
                        BREAK
                ELSE:
                    写入失败计数器 = 0

            日志事件 = 获取下一个待记录事件()

        // 第3步：存储空间检查
        IF internal_state != STATE_FAULT:
            检查剩余存储空间()
            IF 剩余空间 < 20%:
                SET internal_state = STATE_LOW
                执行滚动清理策略()
                向 ad-mcc-01 发送存储空间不足告警

        // 第4步：处理日志查询请求
        IF 收到日志查询请求:
            根据查询条件检索日志()
            构建查询结果()
            向请求方返回日志查询结果

        // 第5步：周期性状态上报
        IF 距上次上报 >= 10秒:
            向 ad-mcc-01 上报存储状态(总容量, 已用, 剩余, 最早时间, 最新时间)

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 存储介质写入失败 | 连续3次失败后切换到内存缓冲区模式，上报告警 | 存储介质恢复可用 |
| 内存缓冲区溢出 | 丢弃最旧的可覆写日志条目，上报告警 | 存储恢复后写入 |
| 存储空间耗尽且无可清理日志 | 发出严重告警，建议更换存储介质或离线导出 | 人工介入清理或更换 |
| 查询请求超时（检索耗时 > 500ms） | 返回部分结果并标记"查询未完成"，建议缩小查询范围 | — |
| 紧急熔断 | 强制将缓冲区内容刷入存储（若可用），暂停接收新日志 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 各类日志事件 | 只读 | 各模块推送 |
| 内部调度总线 | 读 | 日志查询请求 | 只读 | 请求方发送 |
| 内部调度总线 | 写 | 日志查询结果 | 专属写入 | 向请求方返回 |
| 内部调度总线 | 写 | 存储状态上报 | 周期性写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 存储空间不足告警 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 存储故障告警 | 事件触发写入 | 向 ad-mcc-01 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 事故相关的关键日志（异常、故障、严重偏差）必须标记为不可覆写，存储周期 ≥ 3年 |
| S-02 | 日志记录必须包含 UTC 时间戳、模块编号、事件类型等完整元数据 |
| S-03 | 日志存储必须采用追加写模式，禁止修改或删除已落盘的日志条目 |
| S-04 | 本模块仅负责日志的记录、存储与检索，不参与任何车辆控制决策 |
