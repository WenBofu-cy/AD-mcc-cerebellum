# ad-mcc-31-通信总线异常监测单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-31 |
| 模块名称 | 通信总线异常监测单元 |
| 所属分区 | 八、硬件异常应急防护 |
| 核心职责 | 实时监测车辆 CAN 总线、Ethernet 总线（若配备）及各节点心跳信号的通信质量，统计帧错误率、超时帧比例及总线负载率。当通信质量下降至阈值时，触发总线切换（主总线→冗余总线）并向相关模块发出预警；当所有可用总线均出现严重故障时，触发三级降级并请求紧急停车。不参与通信内容解析或路由决策，仅负责通信链路健康监测与故障应急 |
| 依赖模块 | 各 ECU 节点心跳信号（CAN/Ethernet 总线）、网络交换机/网关（提供端口统计信息）、ad-mcc-01（小脑总控调度核心，接收降级触发请求） |
| 被依赖模块 | ad-mcc-28/29/30（各硬件异常监测单元，接收对应子系统通信状态用于综合故障判定）、ad-mcc-01（接收总线健康状态上报）、ad-mcc-38（执行日志记录单元）、ECC-12（通过 CerebellumBus 接收严重通信故障告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常通信 | `NORMAL_COMM` | 所有主总线正常运行，帧错误率在允许范围内 | 系统初始化完成，总线通信正常 |
| 主总线降级 | `PRIMARY_DEGRADED` | 主总线帧错误率或超时帧比例超过预警阈值，已切换至冗余总线 | 主总线异常，冗余总线正常 |
| 冗余总线降级 | `SECONDARY_DEGRADED` | 冗余总线也出现异常，仅剩单总线运行，通信可靠性下降 | 主总线异常且冗余总线亦出现预警 |
| 通信严重故障 | `COMM_CRITICAL` | 所有可用总线均出现严重故障，节点大面积离线，触发三级降级 | 主/冗余总线均严重异常 或 关键节点离线 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| CAN 总线统计（主） | Struct（帧错误率_pct + 超时帧比例_pct + 总线负载率_pct + 总线状态） | CAN 控制器/网关（主总线接口） | 周期性，10Hz | **高** |
| CAN 总线统计（冗余） | Struct（同上） | CAN 控制器/网关（冗余总线接口） | 周期性，10Hz | **高** |
| Ethernet 总线统计（若配备） | Struct（丢包率_pct + 延迟_ms + 链路状态） | 车载以太网交换机 | 周期性，10Hz | **高** |
| 各节点心跳信号 | List（节点ID + 最后心跳时间 + 预期周期 + 在线状态） | 各 ECU 节点（CAN/Ethernet） | 周期性，每个节点独立 | **高** |
| 当前总线拓扑配置 | Struct（主总线ID + 冗余总线ID + 切换策略） | 网络管理模块（系统配置） | 系统初始化加载 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 总线切换指令 | Struct（切换目标总线ID + 切换原因 + 故障总线ID） | CAN/以太网网关控制器 | 主总线故障需切换时 | **高** |
| 通信故障告警 | Struct（故障类型 + 严重等级 + 当前错误率 + 阈值 + 受影响节点列表） | ad-mcc-01（内部调度总线）、ad-mcc-28/29/30（内部调度总线）、ECC-12（严重时通过 CerebellumBus） | 总线异常或节点离线时 | **高** |
| 降级触发请求 | Struct（目标降级等级 + 触发原因） | ad-mcc-01（小脑总控调度核心） | 所有总线严重故障时 | **高** |
| 通信健康状态上报 | Struct（主总线状态 + 冗余总线状态 + 在线节点数/总节点数 + 当前故障等级） | ad-mcc-01、ad-mcc-38 | 周期性，每1秒 | 普通 |
| 通信故障事件记录 | Struct（故障类型 + 发生时间 + 持续时长 + 影响范围） | ad-mcc-38 执行日志记录单元 | 故障发生与恢复时 | 普通 |


## 故障判定阈值

| 故障类型 | 正常范围 | 预警阈值 | 严重阈值 | 致命阈值 | 判定延时 |
|----------|:---:|:---:|:---:|:---:|:---:|
| CAN 帧错误率 | ≤ 0.5% | > 0.5% | > 1% | > 5% 或总线关闭 | 1s 持续 |
| 超时帧比例 | ≤ 1% | > 1% | > 5% | > 10% | 1s 持续 |
| 总线负载率 | ≤ 70% | > 70% | > 85% | > 95% | 1s 持续 |
| 节点心跳超时 | ≤ 预期周期×2 | > 预期周期×3 | > 预期周期×5 | 完全离线 | 每个节点独立判定 |


## 核心处理逻辑

```
FUNCTION bus_fault_monitor_main_loop():
    STATE_NORMAL = NORMAL_COMM
    STATE_PRI_DEG = PRIMARY_DEGRADED
    STATE_SEC_DEG = SECONDARY_DEGRADED
    STATE_CRITICAL = COMM_CRITICAL
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    SET current_active_bus = "主总线"
    初始化故障持续计数器（主总线、冗余总线、节点）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：获取总线统计数据
        主总线统计 = 获取主CAN总线统计()
        冗余总线统计 = 获取冗余CAN总线统计()
        节点心跳列表 = 获取所有节点心跳状态()
        以太网统计 = 获取Ethernet总线统计（若有）

        // 第3步：评估各总线健康状态
        主总线健康 = 评估总线健康(主总线统计)
        冗余总线健康 = 评估总线健康(冗余总线统计)

        // 第4步：统计离线节点
        离线节点列表 = 筛选心跳超时的节点()
        关键节点离线 = 是否存在关键节点（ECC、MCC主控、ESP）离线

        // 第5步：分级处置
        IF 关键节点离线:
            SET internal_state = STATE_CRITICAL
            向 ad-mcc-01 发送降级触发请求(三级降级, "关键节点离线")
            向 ECC-12 发送通信故障告警("致命", "关键节点离线")
            CONTINUE

        IF 主总线健康 == "严重" AND 冗余总线健康 == "严重":
            SET internal_state = STATE_CRITICAL
            向 ad-mcc-01 发送降级触发请求(三级降级, "全部总线严重故障")
            向 ECC-12 发送通信故障告警("致命", "全部总线故障")
            CONTINUE

        IF 主总线健康 == "严重" AND 冗余总线健康 != "严重":
            IF current_active_bus == "主总线":
                向网关控制器发送总线切换指令(冗余总线, "主总线严重故障")
                current_active_bus = "冗余总线"
            SET internal_state = STATE_PRI_DEG
            向 ad-mcc-01 发送通信故障告警("严重", "主总线故障，已切换冗余")
            CONTINUE

        IF 主总线健康 == "预警" OR 冗余总线健康 == "预警":
            IF 主总线健康 == "预警" AND 冗余总线健康 == "正常":
                IF current_active_bus == "主总线":
                    向网关控制器发送总线切换指令(冗余总线, "主总线预警")
                    current_active_bus = "冗余总线"
                SET internal_state = STATE_PRI_DEG
            ELSE IF 主总线健康 == "预警" AND 冗余总线健康 == "预警":
                SET internal_state = STATE_SEC_DEG
                向 ad-mcc-01 发送通信故障告警("预警", "双总线均预警")
            ELSE IF 冗余总线健康 == "预警":
                SET internal_state = STATE_SEC_DEG
                向 ad-mcc-01 发送通信故障告警("预警", "冗余总线预警")
        ELSE:
            IF internal_state != STATE_NORMAL AND internal_state != STATE_PAUSED:
                // 若当前在冗余总线运行且主总线已恢复，可切换回主总线
                IF current_active_bus == "冗余总线" AND 主总线健康 == "正常":
                    向网关控制器发送总线切换指令(主总线, "主总线已恢复")
                    current_active_bus = "主总线"
                SET internal_state = STATE_NORMAL
                向 ad-mcc-01 发送通信故障告警("恢复", "总线通信恢复正常")

        // 第6步：节点离线告警
        IF 离线节点列表 非空:
            向对应子系统监测模块（ad-mcc-28/29/30）发送节点离线通知
            记录离线事件

        // 第7步：周期性状态上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 上报通信健康状态
            向 ad-mcc-38 记录状态快照

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 主总线故障，冗余总线正常 | 立即切换至冗余总线，上报告警 | 主总线恢复正常且持续3秒，可回切 |
| 主总线与冗余总线均故障 | 触发三级降级，禁止自动驾驶，请求紧急停车 | 至少一路总线恢复正常 |
| 关键节点（ECC/ESP）离线 | 立即触发三级降级，标记系统不可信 | 节点恢复在线且心跳稳定 |
| 总线负载率过高（>95%） | 触发降级，请求减少非安全相关通信流量 | 负载率恢复至85%以下 |
| 短暂干扰导致瞬时错误率飙升 | 采用1秒滑动窗口平滑，避免单次毛刺触发误切换 | 滑动窗口内平均错误率下降 |
| 紧急熔断 | 暂停监测，保持最后状态 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| CAN/内部总线 | 读 | 主/冗余 CAN 总线统计 | 只读 | 周期性，10Hz |
| CAN/内部总线 | 读 | Ethernet 总线统计 | 只读 | 周期性，10Hz |
| CAN/内部总线 | 读 | 各节点心跳信号 | 只读 | 周期性 |
| 内部总线 | 读 | 当前总线拓扑配置 | 只读 | 系统初始化 |
| CAN/内部总线 | 写 | 总线切换指令 | 专属写入 | 向网关控制器发送 |
| 内部调度总线 | 写 | 通信故障告警 | 事件触发写入 | 向 ad-mcc-01、ad-mcc-28/29/30 发送 |
| 内部调度总线 | 写 | 降级触发请求 | 事件触发写入 | 向 ad-mcc-01 发送 |
| CerebellumBus | 写 | 严重通信故障告警 | 事件触发写入 | 向 ECC-12 发送 |
| 内部调度总线 | 写 | 通信健康状态上报 | 周期性写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 通信故障事件记录 | 事件触发写入 | 向 ad-mcc-38 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 关键节点（ECC、ESP、MCC主控）离线时，必须立即触发三级降级，通信故障车控系统不可信 |
| S-02 | 总线切换必须在100ms内完成，确保实时控制指令不丢失 |
| S-03 | 所有总线均故障时，车辆必须进入最低风险状态（紧急停车） |
| S-04 | 本模块仅监测通信链路，不解析或修改通信内容 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M31-01 | `NORMAL_COMM`，通信正常 | 主总线错误率=0.2%，负载=50% | 保持正常，无切换 |
| TC-M31-02 | `NORMAL_COMM`，主总线错误率超标 | 主总线错误率=1.5%，冗余正常 | 切换至冗余总线，进入 `PRIMARY_DEGRADED` |
| TC-M31-03 | `PRIMARY_DEGRADED`，主总线恢复 | 主总线错误率=0.3%，持续3秒 | 回切主总线，恢复 `NORMAL_COMM` |
| TC-M31-04 | `NORMAL_COMM`，双总线均严重故障 | 主总线错误率=3%，冗余错误率=4% | 进入 `COMM_CRITICAL`，触发三级降级 |
| TC-M31-05 | 正常通信中，关键节点离线 | ESP 节点心跳超时 | 触发三级降级，上报致命告警 |
| TC-M31-06 | `SYSTEM_PAUSED`，收到熔断 | 紧急熔断指令 | 暂停监测，保持最后状态 |
