# ad-mcc-30-动力系统异常监测单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-30 |
| 模块名称 | 动力系统异常监测单元 |
| 所属分区 | 八、硬件异常应急防护 |
| 核心职责 | 实时监测驱动电机/发动机的输出扭矩响应延迟、温度、逆变器状态及动力系统通信状态，判断动力系统是否存在电气故障、过热、响应滞后或通信失效。当检测到异常时，根据故障严重等级触发分级降级策略：轻微异常时发出预警并限制最大输出功率；严重异常时触发降级，限制功率至50%并限制车速；致命异常时触发三级降级并禁止自动驾驶。不参与动力控制决策，仅做故障诊断与降级触发 |
| 依赖模块 | 驱动电机控制器/发动机 ECU（CAN 总线，提供输出扭矩、响应时间、温度、故障码）、ad-mcc-01（小脑总控调度核心，接收降级触发请求）、ad-mcc-31（通信总线异常监测单元，提供动力系统通信状态） |
| 被依赖模块 | ad-mcc-01（接收异常告警与降级触发）、ad-mcc-09（油门开度解算单元，可参考降级后的功率限制）、ad-mcc-38（执行日志记录单元，记录故障事件）、ECC-12（通过 CerebellumBus 接收严重故障告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监测 | `NORMAL_MONITOR` | 动力系统运行正常，各项指标在容许范围内 | 系统初始化完成，控制器在线且数据有效 |
| 轻微异常预警 | `MINOR_FAULT` | 检测到扭矩响应延迟偏高或温度偏高，暂时不影响控制 | 响应延迟 > 100ms 或 温度 > 140°C，持续 500ms |
| 严重异常降级 | `SEVERE_FAULT` | 扭矩响应严重滞后或温度过高，可能影响动力输出 | 响应延迟 > 200ms 或 温度 > 160°C 或 通信错误率 > 1% |
| 致命异常禁驾 | `CRITICAL_FAULT` | 动力系统完全失效或控制器通信中断，禁止自动驾驶 | 电机/发动机故障停机 或 控制器离线 > 200ms |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 输出扭矩响应数据 | Struct（目标扭矩 + 实际扭矩 + 响应延迟_ms） | CAN 总线（电机/发动机控制器） | 实时，100Hz | **高** |
| 动力系统温度 | Float（°C） | CAN 总线（温度传感器） | 实时，10Hz | **高** |
| 动力系统故障码 | Struct（故障码 + 故障等级 + 描述） | CAN 总线（控制器） | 事件触发 | **高** |
| 控制器在线状态 | Bool（在线/离线） | CAN 总线（节点心跳） | 周期性，10Hz | **高** |
| 动力通信总线状态 | Struct（CAN 帧错误率 + 节点心跳） | ad-mcc-31 通信总线异常监测单元 | 周期性，10Hz | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 动力系统异常告警 | Struct（故障类型 + 严重等级 + 当前值 + 阈值 + 建议动作） | ad-mcc-01（内部调度总线）、ECC-12（严重时通过 CerebellumBus） | 故障触发时 | **高** |
| 降级触发请求 | Struct（目标降级等级 + 触发原因 + 限速值 + 功率限制比例） | ad-mcc-01（小脑总控调度核心） | 严重/致命异常时 | **高** |
| 功率限制指令 | Struct（最大允许功率比例 + 限制原因） | ad-mcc-09 油门开度解算单元 | 轻微/严重异常时 | **高** |
| 动力健康状态上报 | Struct（响应延迟 + 温度 + 控制器在线 + 通信状态 + 当前故障等级） | ad-mcc-01、ad-mcc-38 | 周期性，每1秒 | 普通 |
| 故障事件记录 | Struct（故障类型 + 发生时间 + 当前值 + 持续时长） | ad-mcc-38 执行日志记录单元 | 故障发生与恢复时 | 普通 |


## 故障判定阈值

| 故障类型 | 正常范围 | 预警阈值 | 严重阈值 | 致命阈值 | 判定延时 |
|----------|:---:|:---:|:---:|:---:|:---:|
| 扭矩响应延迟 | ≤ 50ms | > 100ms | > 200ms | > 500ms | 500ms 持续 |
| 动力系统温度 | ≤ 120°C | > 140°C | > 160°C | > 180°C（停机） | 500ms 持续 |
| 控制器在线状态 | 在线 | — | 离线 < 100ms | 离线 > 200ms | 立即/超时判定 |
| 通信帧错误率 | ≤ 0.5% | > 0.5% | > 1% | 总线完全中断 | 1s 持续 |


## 核心处理逻辑

```
FUNCTION powertrain_fault_monitor_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_MINOR = MINOR_FAULT
    STATE_SEVERE = SEVERE_FAULT
    STATE_CRITICAL = CRITICAL_FAULT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化故障持续计数器（延迟、温度、通信）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL
            重置所有故障计数器

        // 第2步：获取传感器数据
        扭矩数据 = 获取扭矩响应数据()
        温度 = 获取动力系统温度()
        故障码 = 获取动力系统故障码()
        控制器在线 = 获取控制器在线状态()
        通信状态 = 获取动力通信总线状态()

        // 第3步：控制器离线判定
        IF NOT 控制器在线:
            离线计时器 += 控制周期
            IF 离线计时器 > 200ms:
                SET internal_state = STATE_CRITICAL
                向 ad-mcc-01 发送降级触发请求(三级降级, "动力控制器离线")
                向 ECC-12 发送动力系统异常告警("致命", "控制器离线")
                CONTINUE
        ELSE:
            离线计时器 = 0

        // 第4步：响应延迟判定
        延迟状态 = "正常"
        IF 扭矩数据.响应延迟 > 200.0:
            延迟状态 = "严重"
            延迟超限计数器++
        ELSE IF 扭矩数据.响应延迟 > 100.0:
            延迟状态 = "预警"
            延迟超限计数器++
        ELSE:
            延迟超限计数器 = 0

        // 第5步：温度判定
        温度状态 = "正常"
        IF 温度 > 180.0:
            温度状态 = "致命"
        ELSE IF 温度 > 160.0:
            温度状态 = "严重"
        ELSE IF 温度 > 140.0:
            温度状态 = "预警"

        // 第6步：通信状态判定
        通信状态_flag = "正常"
        IF 通信状态.帧错误率 > 1.0:
            通信状态_flag = "严重"
        ELSE IF 通信状态.帧错误率 > 0.5:
            通信状态_flag = "预警"

        // 第7步：综合故障等级判定
        IF 温度状态 == "致命" OR 控制器离线 OR 故障码.等级 == "致命":
            SET internal_state = STATE_CRITICAL
            向 ad-mcc-01 发送降级触发请求(三级降级, 故障原因)
        ELSE IF 延迟状态 == "严重" OR 温度状态 == "严重" OR 通信状态_flag == "严重" OR 故障码.等级 == "严重":
            IF 延迟超限计数器 > 50（=500ms）:
                SET internal_state = STATE_SEVERE
                向 ad-mcc-01 发送降级触发请求(二级降级, 限速40km/h, 功率限制50%)
                向 ad-mcc-09 发送功率限制指令(50%)
        ELSE IF 延迟状态 == "预警" OR 温度状态 == "预警" OR 通信状态_flag == "预警":
            IF 延迟超限计数器 > 50:
                SET internal_state = STATE_MINOR
                向 ad-mcc-01 发送动力系统异常告警("预警", 故障原因)
                向 ad-mcc-09 发送功率限制指令(80%)
        ELSE:
            IF internal_state != STATE_NORMAL:
                记录故障恢复日志
                SET internal_state = STATE_NORMAL
                向 ad-mcc-09 发送功率限制指令(100%)
            重置所有故障计数器

        // 第8步：周期性状态上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 上报动力健康状态
            向 ad-mcc-38 记录状态快照

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 控制器短暂离线（< 200ms） | 保持当前状态，使用最后有效扭矩值 | 控制器恢复在线 |
| 控制器离线 > 200ms | 立即触发三级降级，禁止自动驾驶 | 控制器恢复在线且持续1秒 |
| 动力系统过热（> 180°C） | 触发致命故障，立即限制功率为0，请求停车 | 温度降至160°C以下 |
| 扭矩响应严重滞后 | 限制功率至50%，触发二级降级 | 响应延迟恢复正常且持续3秒 |
| 故障恢复后 | 所有指标正常且持续3秒后，解除降级状态 | — |
| 紧急熔断 | 暂停监测，保持最后故障状态 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| CAN 总线 | 读 | 输出扭矩响应数据 | 只读 | 实时 |
| CAN 总线 | 读 | 动力系统温度 | 只读 | 实时 |
| CAN 总线 | 读 | 动力系统故障码 | 只读 | 事件触发 |
| CAN 总线 | 读 | 控制器在线状态 | 只读 | 周期性 |
| 内部调度总线 | 读 | 动力通信总线状态 | 只读 | ad-mcc-31 提供 |
| 内部调度总线 | 写 | 动力系统异常告警 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 降级触发请求 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 功率限制指令 | 事件触发写入 | 向 ad-mcc-09 发送 |
| CerebellumBus | 写 | 严重/致命异常告警 | 事件触发写入 | 向 ECC-12 发送 |
| 内部调度总线 | 写 | 动力健康状态上报 | 周期性写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 故障事件记录 | 事件触发写入 | 向 ad-mcc-38 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 动力控制器离线超过200ms时，必须立即禁止自动驾驶，不得依赖最后有效值 |
| S-02 | 动力系统过热（>180°C）必须立即降低功率至0，防止永久损坏 |
| S-03 | 故障恢复后需持续监测 3 秒确认正常，方可解除降级，防止间歇性故障反复触发 |
| S-04 | 本模块仅做故障诊断与降级触发，不直接干预动力执行器控制 |
