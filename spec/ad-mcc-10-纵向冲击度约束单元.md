# ad-mcc-10-纵向冲击度约束单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-10 |
| 模块名称 | 纵向冲击度约束单元 |
| 所属分区 | 三、动力控制集群 |
| 核心职责 | 接收 ad-mcc-09 输出的原始油门开度序列，基于当前车速、目标加速度与驾驶模式，对油门开度变化率进行约束，确保起步、加速、巡航过程中纵向冲击度始终 ≤ 5m/s³（正常模式）或对应降级模式的更严格上限。对超出冲击度限制的油门指令进行削峰与平滑修正，输出冲击度合规的油门开度序列至 ad-mcc-11。不参与任何场景判断与驾驶决策 |
| 依赖模块 | ad-mcc-09（油门开度解算单元，提供原始油门开度序列）、ad-mcc-01（小脑总控调度核心，下发驾驶模式信号） |
| 被依赖模块 | ad-mcc-11（加速平顺滤波单元，接收冲击度合规的油门开度序列） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常约束 | `NORMAL_CONSTRAINT` | 正常模式下执行纵向冲击度 ≤ 5m/s³ 约束 | 系统初始化完成，驾驶模式=正常 |
| 降级约束 | `DEGRADED_CONSTRAINT` | 降级模式下执行更严格的冲击度上限 | 收到一级/二级降级模式信号 |
| 非铺装约束 | `UNPAVED_CONSTRAINT` | 非铺装模式下执行柔和冲击度上限 | 收到非铺装道路模式信号 |
| 紧急模式 | `EMERGENCY_MODE` | 紧急制动或三级降级，不限制冲击度（安全优先） | 收到紧急制动指令或三级降级信号 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 原始油门开度序列 | Struct（时间戳 + 目标油门开度_pct + 期望加速度_ms2 + 解算方法 + 置信度） | ad-mcc-09 油门开度解算单元 | 每次解算完成后 | **高** |
| 驾驶模式信号 | Enum（正常/一级降级/二级降级/三级降级/非铺装道路） | ad-mcc-01 小脑总控调度核心 | 驾驶模式变更时 | **高** |
| 当前车速 | Float（km/h） | CAN 总线 | 实时，100Hz | **高** |
| 当前实际油门开度 | Float（0–100%） | CAN 总线（油门踏板位置传感器） | 实时，100Hz | **高** |
| 紧急制动指令 | Struct（意图类型=紧急制动 + 冲击度豁免标记=True） | ad-mcc-01 小脑总控调度核心 | 碰撞风险触发时 | **最高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 冲击度合规油门序列 | Struct（时间戳 + 修正后油门开度_pct + 原始油门开度_pct + 是否触发约束 + 约束原因 + 期望加速度_ms2） | ad-mcc-11 加速平顺滤波单元 | 每次约束计算完成后 | **高** |
| 约束触发记录 | Struct（时间戳 + 原始值 + 修正值 + 触发原因 + 当前模式） | ad-mcc-38 执行日志记录单元 | 冲击度约束触发时 | 普通 |


## 各驾驶模式冲击度上限

| 驾驶模式 | 纵向冲击度上限 | 油门开度变化率上限 | 说明 |
|----------|:---:|:---:|------|
| 正常 | 5.0 m/s³ | 无额外限制 | 标准舒适性约束 |
| 一级降级 | 4.0 m/s³ | 15% / 100ms | 适度收紧 |
| 二级降级 | 3.0 m/s³ | 10% / 100ms | 严格限制 |
| 三级降级 | 豁免（不限制） | 豁免 | 安全优先，允许最大制动 |
| 非铺装道路 | 3.0 m/s³ | 8% / 100ms | 柔和操控，防止打滑 |
| 紧急制动 | 豁免（不限制） | 豁免 | 安全优先，允许全力制动 |


## 核心处理逻辑

```
FUNCTION longitudinal_jerk_constraint_main_loop():
    STATE_NORMAL = NORMAL_CONSTRAINT
    STATE_DEGRADED = DEGRADED_CONSTRAINT
    STATE_UNPAVED = UNPAVED_CONSTRAINT
    STATE_EMERGENCY = EMERGENCY_MODE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    SET current_jerk_limit = 5.0  // 默认正常模式
    SET current_throttle_rate_limit = None  // 无额外限制

    // 历史值（用于冲击度计算）
    SET prev_target_accel = 0.0
    SET prev_timestamp = 0.0
    SET prev_throttle_pct = 0.0

    WHILE 系统运行中:
        // 第1步：最高优先级——紧急制动豁免
        IF 收到紧急制动指令:
            SET internal_state = STATE_EMERGENCY
            冲击度豁免 = True
            // 直接放行，不做约束
            IF 收到原始油门开度序列:
                向 ad-mcc-11 发送冲击度合规油门序列(
                    修正后油门开度_pct = 原始油门开度,
                    是否触发约束 = False,
                    约束原因 = "紧急制动豁免"
                )
            CONTINUE

        // 第2步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第3步：驾驶模式切换
        IF 收到驾驶模式信号:
            SWITCH 驾驶模式:
                CASE "正常":
                    SET internal_state = STATE_NORMAL
                    SET current_jerk_limit = 5.0
                    SET current_throttle_rate_limit = None
                CASE "一级降级":
                    SET internal_state = STATE_DEGRADED
                    SET current_jerk_limit = 4.0
                    SET current_throttle_rate_limit = 15  // %/100ms
                CASE "二级降级":
                    SET internal_state = STATE_DEGRADED
                    SET current_jerk_limit = 3.0
                    SET current_throttle_rate_limit = 10
                CASE "三级降级":
                    SET internal_state = STATE_EMERGENCY
                    SET current_jerk_limit = None  // 豁免
                CASE "非铺装道路":
                    SET internal_state = STATE_UNPAVED
                    SET current_jerk_limit = 3.0
                    SET current_throttle_rate_limit = 8

        // 第4步：接收原始油门开度序列
        IF 收到 ad-mcc-09 原始油门开度序列:
            原始油门 = 序列.目标油门开度_pct
            期望加速度 = 序列.期望加速度_ms2
            当前时间戳 = 序列.时间戳

            // 紧急模式直接放行
            IF internal_state == STATE_EMERGENCY:
                向 ad-mcc-11 发送冲击度合规油门序列(
                    修正后油门开度_pct = 原始油门,
                    是否触发约束 = False,
                    约束原因 = "紧急模式豁免"
                )
                CONTINUE

            // 4a. 计算当前冲击度
            时间间隔 = 当前时间戳 - prev_timestamp
            IF 时间间隔 > 0 AND prev_timestamp > 0:
                加速度变化量 = 期望加速度 - prev_target_accel
                当前冲击度 = ABS(加速度变化量) / 时间间隔

                // 4b. 冲击度超限判定
                约束触发 = False
                约束原因 = ""

                IF 当前冲击度 > current_jerk_limit:
                    约束触发 = True
                    约束原因 = f"纵向冲击度超限 ({当前冲击度:.2f} > {current_jerk_limit})"

                    // 计算允许的最大加速度变化量
                    允许加速度变化量 = current_jerk_limit × 时间间隔
                    修正后期望加速度 = prev_target_accel + SIGN(加速度变化量) × 允许加速度变化量

                    // 反推修正后的油门开度
                    // 简化映射：加速度与油门开度近似线性关系
                    IF 期望加速度 != 0 AND prev_target_accel != 期望加速度:
                        油门修正比例 = 修正后期望加速度 / 期望加速度
                        修正后油门 = prev_throttle_pct + (原始油门 - prev_throttle_pct) × 油门修正比例
                    ELSE:
                        修正后油门 = 原始油门
                ELSE:
                    修正后油门 = 原始油门

                // 4c. 油门变化率额外约束（降级/非铺装模式）
                IF current_throttle_rate_limit IS NOT None:
                    油门变化量 = ABS(修正后油门 - prev_throttle_pct)
                    允许变化量 = current_throttle_rate_limit × (时间间隔 / 100ms)
                    IF 油门变化量 > 允许变化量:
                        约束触发 = True
                        约束原因 += IF 约束原因 != "" THEN " + 油门变化率超限" ELSE "油门变化率超限"
                        修正后油门 = prev_throttle_pct + SIGN(修正后油门 - prev_throttle_pct) × 允许变化量
            ELSE:
                修正后油门 = 原始油门

            // 4d. 更新历史值
            prev_target_accel = 期望加速度
            prev_throttle_pct = 修正后油门
            prev_timestamp = 当前时间戳

            // 4e. 输出约束后油门序列
            向 ad-mcc-11 发送冲击度合规油门序列(
                时间戳=当前时间戳,
                修正后油门开度_pct=修正后油门,
                原始油门开度_pct=原始油门,
                是否触发约束=约束触发,
                约束原因=约束原因,
                期望加速度_ms2=期望加速度
            )

            // 4f. 约束触发时记录日志
            IF 约束触发:
                向 ad-mcc-38 发送约束触发记录(
                    时间戳=当前时间戳,
                    原始值=原始油门,
                    修正值=修正后油门,
                    触发原因=约束原因,
                    当前模式=internal_state
                )

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 历史值未初始化（首帧） | 直接放行，不执行冲击度计算 | 第二帧起正常计算 |
| 时间间隔异常（≤0 或 >1s） | 视为首帧，重置历史值，直接放行 | 下一帧正常计算 |
| 期望加速度突变（相邻帧差值 > 5m/s²） | 视为传感器噪声或通信抖动，自动削峰至允许最大变化量 | 连续3帧正常后恢复标准约束 |
| 紧急制动指令 | 冲击度豁免，直接放行，不约束 | 紧急制动解除后恢复当前模式约束 |
| 原始油门开度置信度 < 0.7 | 收紧冲击度上限20%，增加安全裕度 | 置信度恢复 ≥ 0.7 |
| 紧急熔断 | 暂停约束，维持当前修正参数 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 紧急制动指令为最高优先级，收到后冲击度约束立即豁免，油门指令无条件直接放行 |
| S-02 | 本模块仅约束冲击度与油门变化率，不改变油门开度的终值方向（加速/减速意图不变） |
| S-03 | 冲击度约束不得导致油门响应完全停滞。修正后的加速度变化量不得为0（除非目标加速度本身为0） |
| S-04 | 原始油门开度置信度低于0.7时，必须收紧约束上限20%，增加安全裕度 |
| S-05 | 本模块仅做冲击度约束修正，不参与任何场景判断与驾驶决策 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M10-01 | `NORMAL_CONSTRAINT`，正常冲击度 | 目标油门=30%，期望加速度=1.5m/s²，上一帧加速度=1.0m/s²，间隔=100ms | 冲击度=5.0（恰好不超限），放行 |
| TC-M10-02 | `NORMAL_CONSTRAINT`，冲击度超限 | 目标油门=50%，期望加速度=3.0m/s²，上一帧加速度=0.5m/s²，间隔=100ms | 冲击度=25 > 5，触发约束，修正加速度至1.0m/s²，修正油门 |
| TC-M10-03 | `DEGRADED_CONSTRAINT`（二级降级） | 目标油门=40%，期望加速度=2.0m/s²，上一帧加速度=1.0m/s²，间隔=100ms | 冲击度=10 > 3，触发约束，修正至限值内 + 油门变化率 ≤ 10%/100ms |
| TC-M10-04 | `EMERGENCY_MODE`，紧急制动 | 目标油门=0%，期望加速度=-6.0m/s²，紧急制动豁免 | 直接放行，不约束 |
| TC-M10-05 | 首帧（历史值未初始化） | 目标油门=35%，无历史值 | 直接放行，不计算冲击度 |
| TC-M10-06 | 原始油门置信度 < 0.7 | 目标油门=40%，置信度=0.5 | 冲击度上限收紧20%（5.0→4.0），以收紧后限值约束 |
