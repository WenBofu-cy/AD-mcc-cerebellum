# ad-mcc-29-制动系统异常监测单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-29 |
| 模块名称 | 制动系统异常监测单元 |
| 所属分区 | 八、硬件异常应急防护 |
| 核心职责 | 实时监测制动主缸压力传感器双路信号一致性、制动液位、制动踏板开关状态及制动系统通信状态，判断制动系统是否存在传感器故障、液位过低、管路泄漏或通信失效。当检测到异常时，根据故障严重等级触发分级降级策略：轻微异常时发出预警并限制制动压力上限；严重异常时触发降级，限制车速至30km/h并优先使用再生制动；致命异常时触发三级降级并禁止自动驾驶。不参与制动控制决策，仅做故障诊断与降级触发 |
| 依赖模块 | 制动主缸压力传感器（CAN 总线，双路冗余）、制动液位传感器（CAN 总线）、制动踏板开关（CAN 总线）、ad-mcc-01（小脑总控调度核心，接收降级触发请求）、ad-mcc-31（通信总线异常监测单元，提供制动系统通信状态） |
| 被依赖模块 | ad-mcc-01（接收异常告警与降级触发）、ad-mcc-13（制动压力解算单元，可参考降级后的压力上限）、ad-mcc-17（再生制动优先协调单元，严重故障时优先再生制动）、ad-mcc-38（执行日志记录单元，记录故障事件）、ECC-12（通过 CerebellumBus 接收严重故障告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监测 | `NORMAL_MONITOR` | 制动系统运行正常，各项指标在容许范围内 | 系统初始化完成，传感器在线且数据有效 |
| 轻微异常预警 | `MINOR_FAULT` | 检测到制动液位偏低或压力传感器偏差接近阈值，暂时不影响控制 | 液位 < 80% 或 双路压力偏差 > 0.3 MPa，持续 500ms |
| 严重异常降级 | `SEVERE_FAULT` | 制动液位过低或压力传感器偏差过大，可能影响制动力控制精度 | 液位 < 50% 或 双路压力偏差 > 0.5 MPa 或 通信错误率 > 1% |
| 致命异常禁驾 | `CRITICAL_FAULT` | 制动系统完全失效或双路传感器均故障，禁止自动驾驶 | 液位 < 20% 或 双路压力传感器同时失效 或 制动踏板开关失效 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 制动主缸压力（主路） | Float（MPa） | CAN 总线（制动压力传感器主路） | 实时，200Hz | **高** |
| 制动主缸压力（冗余路） | Float（MPa） | CAN 总线（制动压力传感器冗余路） | 实时，200Hz | **高** |
| 制动液位 | Float（0–100%） | CAN 总线（制动液位传感器） | 周期性，1Hz | **高** |
| 制动踏板开关状态 | Bool（踩下/释放） | CAN 总线（制动踏板开关） | 实时，100Hz | **高** |
| 制动系统故障码 | Struct（故障码 + 故障等级 + 描述） | CAN 总线（ESP/ABS 控制器） | 事件触发 | **高** |
| 制动通信总线状态 | Struct（CAN 帧错误率 + 节点心跳） | ad-mcc-31 通信总线异常监测单元 | 周期性，10Hz | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 制动系统异常告警 | Struct（故障类型 + 严重等级 + 当前值 + 阈值 + 建议动作） | ad-mcc-01（内部调度总线）、ECC-12（严重时通过 CerebellumBus） | 故障触发时 | **高** |
| 降级触发请求 | Struct（目标降级等级 + 触发原因 + 限速值 + 制动压力上限 + 再生制动优先标记） | ad-mcc-01（小脑总控调度核心） | 严重/致命异常时 | **高** |
| 制动压力上限限制 | Struct（最大允许压力_MPa + 限制原因） | ad-mcc-13 制动压力解算单元 | 轻微/严重异常时 | **高** |
| 制动健康状态上报 | Struct（主路压力 + 冗余压力 + 压力偏差 + 液位 + 通信状态 + 当前故障等级） | ad-mcc-01、ad-mcc-38 | 周期性，每1秒 | 普通 |
| 故障事件记录 | Struct（故障类型 + 发生时间 + 当前值 + 持续时长） | ad-mcc-38 执行日志记录单元 | 故障发生与恢复时 | 普通 |


## 故障判定阈值

| 故障类型 | 正常范围 | 预警阈值 | 严重阈值 | 致命阈值 | 判定延时 |
|----------|:---:|:---:|:---:|:---:|:---:|
| 主缸压力双路偏差 | ≤ 0.2 MPa | > 0.3 MPa | > 0.5 MPa | 双路均失效 | 500ms 持续 |
| 制动液位 | ≥ 90% | < 80% | < 50% | < 20% | 立即/1s持续 |
| 制动踏板开关 | 正常触发 | 偶发不触发 | 频繁不触发 | 完全失效 | 500ms 持续 |
| 通信帧错误率 | ≤ 0.5% | > 0.5% | > 1% | 总线完全中断 | 1s 持续 |
| 传感器心跳超时 | ≤ 50ms | > 50ms | > 100ms | > 200ms | 立即/超时判定 |


## 核心处理逻辑

```
FUNCTION brake_fault_monitor_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_MINOR = MINOR_FAULT
    STATE_SEVERE = SEVERE_FAULT
    STATE_CRITICAL = CRITICAL_FAULT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化故障持续计数器（压力、液位、踏板、通信）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL
            重置所有故障计数器

        // 第2步：获取传感器数据
        主路压力 = 获取主缸压力主路()
        冗余压力 = 获取主缸压力冗余()
        液位 = 获取制动液位()
        踏板开关 = 获取制动踏板开关状态()
        故障码 = 获取制动系统故障码()
        通信状态 = 获取制动通信总线状态()

        // 第3步：传感器健康判定
        主路有效 = NOT 主路压力超时
        冗余有效 = NOT 冗余压力超时
        双路失效 = NOT 主路有效 AND NOT 冗余有效

        IF 双路失效:
            SET internal_state = STATE_CRITICAL
            向 ad-mcc-01 发送降级触发请求(三级降级, "制动压力传感器双路失效", 限速=0)
            向 ECC-12 发送制动系统异常告警("致命", "传感器双路失效")
            CONTINUE

        IF NOT 主路有效 OR NOT 冗余有效:
            使用有效的一路作为实际压力
            标记传感器部分故障
        ELSE:
            压力偏差 = ABS(主路压力 - 冗余压力)

        // 第4步：压力偏差判定
        压力状态 = "正常"
        IF 压力偏差 > 0.5:
            压力状态 = "严重"
            压力超限计数器++
        ELSE IF 压力偏差 > 0.3:
            压力状态 = "预警"
            压力超限计数器++
        ELSE:
            压力超限计数器 = 0

        // 第5步：液位判定
        液位状态 = "正常"
        IF 液位 < 20.0:
            液位状态 = "致命"
        ELSE IF 液位 < 50.0:
            液位状态 = "严重"
        ELSE IF 液位 < 80.0:
            液位状态 = "预警"

        // 第6步：踏板开关判定
        踏板状态 = "正常"
        IF 踏板开关.完全失效:
            踏板状态 = "致命"
        ELSE IF 踏板开关.频繁不触发:
            踏板状态 = "严重"
        ELSE IF 踏板开关.偶发不触发:
            踏板状态 = "预警"

        // 第7步：通信状态判定
        通信状态_flag = "正常"
        IF 通信状态.帧错误率 > 1.0:
            通信状态_flag = "严重"
        ELSE IF 通信状态.帧错误率 > 0.5:
            通信状态_flag = "预警"

        // 第8步：综合故障等级判定
        IF 压力状态 == "严重" OR 液位状态 == "严重" OR 液位状态 == "致命" OR 踏板状态 == "致命" OR 通信状态_flag == "严重" OR 故障码.等级 >= 严重:
            IF 液位状态 == "致命" OR 踏板状态 == "致命":
                SET internal_state = STATE_CRITICAL
                向 ad-mcc-01 发送降级触发请求(三级降级, 故障原因)
            ELSE IF 压力超限计数器 > 100（=500ms）:
                SET internal_state = STATE_SEVERE
                向 ad-mcc-01 发送降级触发请求(二级降级, 限速30km/h, 制动压力上限=5MPa, 再生制动优先)
                向 ad-mcc-13 发送制动压力上限限制(5.0 MPa)
                向 ad-mcc-17 发送再生制动优先标记
        ELSE IF 压力状态 == "预警" OR 液位状态 == "预警" OR 踏板状态 == "预警" OR 通信状态_flag == "预警":
            IF 压力超限计数器 > 100:
                SET internal_state = STATE_MINOR
                向 ad-mcc-01 发送制动系统异常告警("预警", 故障原因)
                向 ad-mcc-13 发送制动压力上限限制(8.0 MPa)
        ELSE:
            IF internal_state != STATE_NORMAL:
                记录故障恢复日志
                SET internal_state = STATE_NORMAL
                向 ad-mcc-13 发送取消制动压力限制
                向 ad-mcc-17 发送取消再生制动优先
            重置所有故障计数器

        // 第9步：周期性状态上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 上报制动健康状态
            向 ad-mcc-38 记录状态快照

        SLEEP 5ms  // 200Hz，对齐制动压力传感器频率
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 制动压力传感器单路失效 | 使用有效的一路继续工作，标记传感器降级，上报告警 | 失效路恢复 |
| 制动压力传感器双路失效 | 立即触发三级降级，制动压力由再生制动和最后有效值估算，禁止自动驾驶 | 至少一路恢复 |
| 制动液位过低（< 20%） | 触发致命故障，立即禁止自动驾驶，请求紧急停车 | 液位恢复正常 |
| 制动踏板开关失效 | 触发致命故障，无法确认驾驶员制动意图，禁止自动驾驶 | 踏板开关恢复 |
| 故障恢复后 | 所有指标正常且持续3秒后，解除降级状态 | — |
| 紧急熔断 | 暂停监测，保持最后故障状态 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 制动压力传感器双路失效时，必须立即禁止自动驾驶，不得依赖单路或历史数据 |
| S-02 | 制动液位过低（<20%）时，视为制动系统严重故障，必须立即停车 |
| S-03 | 制动踏板开关失效时，无法确认驾驶员制动意图，禁止自动驾驶 |
| S-04 | 故障恢复后需持续监测 3 秒确认正常，方可解除降级，防止间歇性故障反复触发 |
| S-05 | 本模块仅做故障诊断与降级触发，不直接干预制动执行器控制 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M29-01 | `NORMAL_MONITOR`，所有指标正常 | 压力偏差=0.1MPa，液位=95% | 保持正常，无告警 |
| TC-M29-02 | `NORMAL_MONITOR`，液位偏低 | 液位=70%，持续 | 进入 `MINOR_FAULT`，发送预警告警，限制制动压力上限=8.0MPa |
| TC-M29-03 | `NORMAL_MONITOR`，压力偏差过大 | 双路偏差=0.6MPa，持续600ms | 进入 `SEVERE_FAULT`，触发二级降级，限速30km/h，再生优先 |
| TC-M29-04 | `NORMAL_MONITOR`，液位致命 | 液位=15% | 进入 `CRITICAL_FAULT`，触发三级降级，禁止自动驾驶 |
| TC-M29-05 | `NORMAL_MONITOR`，双路传感器失效 | 主路超时200ms，冗余超时200ms | 进入 `CRITICAL_FAULT`，触发三级降级 |
| TC-M29-06 | `SEVERE_FAULT`，故障恢复 | 所有指标正常，持续3秒 | 恢复 `NORMAL_MONITOR`，解除所有限制 |
