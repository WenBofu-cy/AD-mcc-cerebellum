# ad-mcc-11-加速平顺滤波单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-11 |
| 模块名称 | 加速平顺滤波单元 |
| 所属分区 | 三、动力控制集群 |
| 核心职责 | 接收 ad-mcc-10 输出的冲击度合规油门开度序列，对油门指令进行低通滤波与平滑处理，消除残余的高频波动与微幅抖动，确保油门开度变化曲线连续、圆滑、无阶跃感。输出最终的油门踏板目标开度指令至车辆底层驱动接口（或 ad-mcc-12 供偏差监控）。不参与任何场景判断与驾驶决策 |
| 依赖模块 | ad-mcc-10（纵向冲击度约束单元，提供冲击度合规油门序列）、ad-mcc-01（小脑总控调度核心，下发驾驶模式信号以调整滤波强度） |
| 被依赖模块 | ad-mcc-12（动力执行偏差监控单元，接收最终油门指令作为目标值进行偏差比对）；实际底层油门执行器（通过 CAN 总线或内部接口下发） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常滤波 | `NORMAL_FILTER` | 标准滤波强度，适用于铺装路面正常驾驶 | 系统初始化完成，驾驶模式=正常 |
| 柔和滤波 | `SOFT_FILTER` | 增强滤波，平滑度优先，适用于非铺装路面或舒适模式 | 收到非铺装道路模式信号或用户设定为舒适风格 |
| 快速响应 | `FAST_RESPONSE` | 降低滤波强度，允许更快油门响应（如运动模式） | 用户设定为高效通行/运动风格 |
| 冻结保持 | `HOLD` | 紧急情况下冻结当前滤波状态，保持当前油门输出，不响应新指令变化 | 收到紧急制动、碰撞后响应或三级降级指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 滤波参数配置

### 一、各驾驶模式/风格下的滤波参数

| 模式/风格 | 滤波类型 | 截止频率 (Hz) | 平滑系数 α | 最大滤波延迟 | 油门变化率柔化 |
|----------|:---:|:---:|:---:|:---:|:---:|
| 正常 + 标准通勤 | 一阶低通 | 8.0 | 0.2 | < 30ms | 无额外限制 |
| 正常 + 平顺舒适 | 一阶低通 | 5.0 | 0.35 | < 50ms | 每帧最大变化 3% |
| 正常 + 高效通行 | 一阶低通 | 12.0 | 0.1 | < 20ms | 无额外限制 |
| 一级降级 | 一阶低通 | 6.0 | 0.3 | < 40ms | 每帧最大变化 4% |
| 二级降级 | 一阶低通 | 4.0 | 0.4 | < 60ms | 每帧最大变化 2.5% |
| 非铺装道路 | 一阶低通 + 移动平均(3帧) | 3.0 | 0.5 | < 80ms | 每帧最大变化 2% |
| 紧急/冻结 | 无滤波 | 直通 | — | 0 | 输出锁定值 |

### 二、一阶低通滤波公式

```
filtered_throttle = α × raw_throttle + (1 - α) × prev_filtered_throttle

其中:
  α = 平滑系数（0.0–1.0，越小越平滑，延迟越大）
  等效截止频率 f_c = α / (2π × T_s)   (T_s 为采样周期 0.01s)
```


## 核心处理逻辑

```
FUNCTION throttle_smooth_filter_main_loop():
    STATE_NORMAL = NORMAL_FILTER
    STATE_SOFT = SOFT_FILTER
    STATE_FAST = FAST_RESPONSE
    STATE_HOLD = HOLD
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    SET current_alpha = 0.2
    SET max_frame_change_pct = None
    SET frozen_throttle = 0.0

    // 历史值
    SET prev_filtered = 0.0
    SET filter_mode_name = "一阶低通 (正常)"

    WHILE 系统运行中:
        // 第1步：最高优先级——冻结保持
        IF 收到紧急状态冻结指令:
            SET internal_state = STATE_HOLD
            frozen_throttle = 指令.目标油门锁定值（若未指定则保持当前滤波值）
            // 持续输出冻结值
            最终油门 = frozen_throttle
            向输出接口发送最终油门目标指令(滤波后=最终油门, 滤波方法="冻结保持")
            CONTINUE

        // 第2步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第3步：滤波参数更新（驾驶模式或风格变化）
        当前模式 = 获取驾驶模式()
        当前风格 = 获取用户驾驶风格设定()

        // 根据组合确定参数
        IF 当前模式 == "非铺装道路":
            SET internal_state = STATE_SOFT
            SET current_alpha = 0.5
            SET max_frame_change_pct = 2.0
            SET filter_mode_name = "一阶低通+移动平均 (非铺装)"
        ELSE IF 当前模式 == "一级降级":
            SET current_alpha = 0.3
            SET max_frame_change_pct = 4.0
            SET filter_mode_name = "一阶低通 (降级)"
        ELSE IF 当前模式 == "二级降级":
            SET current_alpha = 0.4
            SET max_frame_change_pct = 2.5
            SET filter_mode_name = "一阶低通 (降级)"
        ELSE IF 当前模式 == "三级降级":
            // 三级降级通常油门归零，但若需要特定值，则冻结
            SET internal_state = STATE_HOLD
            frozen_throttle = 0.0
            CONTINUE
        ELSE IF 当前风格 == "平顺舒适":
            SET internal_state = STATE_SOFT
            SET current_alpha = 0.35
            SET max_frame_change_pct = 3.0
            SET filter_mode_name = "一阶低通 (舒适)"
        ELSE IF 当前风格 == "高效通行":
            SET internal_state = STATE_FAST
            SET current_alpha = 0.1
            SET max_frame_change_pct = None
            SET filter_mode_name = "一阶低通 (高效)"
        ELSE:
            // 正常 + 标准
            SET internal_state = STATE_NORMAL
            SET current_alpha = 0.2
            SET max_frame_change_pct = None
            SET filter_mode_name = "一阶低通 (标准)"

        // 第4步：接收冲击度合规油门序列
        IF 收到 ad-mcc-10 油门序列:
            raw_throttle = 序列.修正后油门开度_pct
            原始油门 = 序列.原始油门开度_pct

            // 4a. 一阶低通滤波
            filtered = current_alpha × raw_throttle + (1 - current_alpha) × prev_filtered

            // 4b. 非铺装模式额外移动平均（窗口=3）
            IF current_alpha == 0.5:  // 非铺装
                // 获取最近3帧原始值
                raw_window = 获取最近3帧 raw_throttle
                filtered = (raw_window[0] + raw_window[1] + raw_window[2]) / 3.0
                // 然后再低通一次
                filtered = current_alpha × filtered + (1 - current_alpha) × prev_filtered

            // 4c. 帧间变化率限制（若配置）
            IF max_frame_change_pct IS NOT None:
                变化量 = filtered - prev_filtered
                IF ABS(变化量) > max_frame_change_pct:
                    filtered = prev_filtered + SIGN(变化量) × max_frame_change_pct

            // 4d. 边界裁剪
            filtered = CLAMP(filtered, 0.0, 100.0)

            // 4e. 更新历史值
            prev_filtered = filtered

            // 4f. 输出最终油门指令
            最终指令 = {
                时间戳: NOW(),
                滤波后油门开度_pct: filtered,
                原始油门开度_pct: 原始油门,
                滤波方法: filter_mode_name,
                滤波强度: current_alpha
            }
            向 ad-mcc-12 及底层油门执行器发送最终油门目标指令

        // 第5步：周期性滤波状态上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 发送滤波状态上报(
                当前滤波模式=filter_mode_name,
                当前截止频率= current_alpha / (2π × 0.01),
                当前平滑系数=current_alpha,
                滤波延迟_ms=当前滤波延迟估计
            )

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 冲击度合规油门序列超时未收到（>200ms） | 维持上一帧滤波值，标记“油门指令超时”，并缓慢衰减至怠速开度（每秒衰减 5%） | 指令恢复更新 |
| 紧急状态冻结指令 | 锁定当前油门开度或指定值，忽略后续输入，持续输出冻结值 | 紧急状态解除且收到恢复指令 |
| 滤波后开度超出 [0,100] | 强制裁剪至边界 | — |
| 用户驾驶风格信号丢失 | 默认回退为“标准通勤”参数 | 信号恢复 |
| 非铺装模式下传感器剧烈抖动导致油门频繁波动 | 移动平均窗口扩大至5帧，同时平滑系数进一步提高至0.6 | 抖动平息后恢复默认非铺装参数 |
| 紧急熔断 | 暂停滤波计算，保持当前输出值 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 紧急冻结指令为最高优先级，一旦激活，必须锁定当前油门或指定值，严禁响应任何新油门指令 |
| S-02 | 滤波后的油门开度必须严格在 [0, 100] 范围内，不得输出负值或超限值 |
| S-03 | 滤波算法不得引入超过 100ms 的延迟（从输入到输出），确保油门响应及时性 |
| S-04 | 在三级降级或碰撞后，油门必须归零或锁定为 0，不允许滤波算法输出非零值 |
| S-05 | 本模块仅负责油门信号的平滑处理，不改变驾驶意图的方向（加速/减速） |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M11-01 | `NORMAL_FILTER`，α=0.2，历史值=25% | 输入油门=30% | 滤波后 = 0.2×30 + 0.8×25 = 26%，平滑过渡 |
| TC-M11-02 | `SOFT_FILTER`（舒适），α=0.35，历史值=20%，帧变化限制 3% | 输入油门=50% | 滤波后变化不超过 3%，本帧输出 23%，逐步逼近 |
| TC-M11-03 | `FAST_RESPONSE`（高效），α=0.1 | 输入油门=40%，历史=35% | 滤波后 = 0.1×40+0.9×35=35.5%，响应较快 |
| TC-M11-04 | `HOLD`，收到冻结指令，锁定值为 15% | 后续输入任意值 | 持续输出 15%，不随输入变化 |
| TC-M11-05 | 输入超过100% | 输入油门=110% | 滤波后裁剪为 100% |
| TC-M11-06 | 非铺装模式，移动平均窗口3帧 | 最近3帧输入 [30, 35, 40] | 先平均得35%，再经低通滤波输出 |
