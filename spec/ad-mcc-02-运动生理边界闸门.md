# ad-mcc-02-运动生理边界闸门 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-02 |
| 模块名称 | 运动生理边界闸门 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 锁定车辆动力学绝对极限（最大方向盘转角速率、最大纵向加减速度、最大横向加速度、侧翻临界阈值、最高车速等），作为 MCC 运动小脑执行任何操控指令前的最后一道硬约束校验。接收 ad-mcc-01 分发的操控指令或模式切换参数集，逐项校验是否超出物理极限或法规硬约束。超限指令立即拦截，并返回超限原因与修正建议。不参与任何场景判断与驾驶决策 |
| 依赖模块 | ad-mcc-01（小脑总控调度核心，下发待校验的操控指令）、ad-mcc-32（车辆尺寸参数管理单元，提供车宽/轴距/质心高度等基准数据）、ad-mcc-34（动力与制动参数管理单元，提供最大加减速度/最大功率等动力极限） |
| 被依赖模块 | ad-mcc-01（接收校验结果）、ad-mcc-04 至 ad-mcc-38（全部执行模块的指令均须经本模块校验后方可执行） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常校验 | `NORMAL_GATE` | 正常执行边界校验，以标准模式参数为基准 | 系统初始化完成，驾驶模式=正常 |
| 降级校验 | `DEGRADED_GATE` | 以更严格的降级边界执行校验 | 收到一级或二级降级模式信号 |
| 紧急校验 | `EMERGENCY_GATE` | 紧急模式下放宽部分边界（如制动减速度允许到极限），但仍死守安全红线 | 收到三级降级或紧急制动指令 |
| 非铺装校验 | `UNPAVED_GATE` | 以非铺装柔和操控参数为基准执行校验 | 收到非铺装道路模式信号 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断，拒绝所有非安全指令 | 收到紧急熔断指令 |

|--------|----------|----------|----------|:---:|

## 核心处理逻辑

```
FUNCTION boundary_gate_main_loop():
    // 状态标识常量
    STATE_NORMAL = NORMAL_GATE
    STATE_DEGRADED = DEGRADED_GATE
    STATE_EMERGENCY = EMERGENCY_GATE
    STATE_UNPAVED = UNPAVED_GATE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化当前边界参数集 = 加载正常模式边界参数
    初始化车辆物理极限 = 加载车辆出厂标定参数

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：执行模式切换
        IF 收到执行模式切换信号:
            SWITCH 新模式:
                CASE "正常":
                    SET internal_state = STATE_NORMAL
                    当前边界参数集 = 加载正常模式边界参数
                CASE "一级降级":
                CASE "二级降级":
                    SET internal_state = STATE_DEGRADED
                    当前边界参数集 = 加载降级模式边界参数(降级等级)
                CASE "三级降级":
                    SET internal_state = STATE_EMERGENCY
                    当前边界参数集 = 加载紧急模式边界参数
                CASE "非铺装道路":
                    SET internal_state = STATE_UNPAVED
                    当前边界参数集 = 加载非铺装模式边界参数
            // 确认切换完成
            向 ad-mcc-01 返回边界参数更新确认

        // 第3步：接收待校验操控指令
        IF 收到待校验操控指令:
            IF internal_state == STATE_PAUSED:
                向 ad-mcc-01 返回校验拦截信号（原因=系统熔断）
                CONTINUE

            // 3a. 根据指令类型逐项校验
            超限列表 = []
            修正建议 = {}

            CASE 指令.类型 OF:
                "转向":
                    // 校验转角速率
                    目标转角速率 = 指令.目标转角速率
                    IF ABS(目标转角速率) > 当前边界参数集.方向盘转角速率上限:
                        超限列表.append("转角速率超限")
                        修正建议.转角速率 = SIGN(目标转角速率) × 当前边界参数集.方向盘转角速率上限
                    // 湿滑路面额外限制
                    IF 当前路面摩擦系数 < 0.4:
                        允许最大转角速率 = 当前边界参数集.方向盘转角速率上限 × 0.6
                        IF ABS(目标转角速率) > 允许最大转角速率:
                            超限列表.append("湿滑路面转角速率超限")
                            修正建议.转角速率 = SIGN(目标转角速率) × 允许最大转角速率

                "加速":
                    目标加速度 = 指令.目标加速度
                    // 校验纵向加速度
                    IF 目标加速度 > 当前边界参数集.最大纵向加速度:
                        超限列表.append("纵向加速度超限")
                        修正建议.加速度 = 当前边界参数集.最大纵向加速度
                    // 湿滑路面修正
                    加速度上限 = 当前边界参数集.最大纵向加速度 × 获取路面修正系数(当前摩擦系数, "加速")
                    IF 目标加速度 > 加速度上限:
                        超限列表.append(f"湿滑路面加速度超限（上限={加速度上限}）")
                        修正建议.加速度 = 加速度上限

                "制动":
                    目标减速度 = 指令.目标减速度
                    制动类型 = 指令.制动类型
                    IF 制动类型 == "紧急":
                        // 紧急制动允许到物理极限，但仍受路面限制
                        制动上限 = 车辆物理极限.最大紧急制动减速度 × 获取路面修正系数(当前摩擦系数, "制动")
                        IF 目标减速度 > 制动上限:
                            超限列表.append("紧急制动减速度超出路面附着极限")
                            修正建议.减速度 = 制动上限
                    ELSE:
                        // 非紧急制动受模式边界限制
                        IF 目标减速度 > 当前边界参数集.最大制动减速度:
                            超限列表.append("制动减速度超限")
                            修正建议.减速度 = 当前边界参数集.最大制动减速度

                "车速":
                    目标车速 = 指令.目标车速
                    IF 目标车速 > 当前边界参数集.最高车速限制:
                        超限列表.append("车速超限")
                        修正建议.车速 = 当前边界参数集.最高车速限制
                    // 物理极限车速（出厂标定）
                    IF 目标车速 > 车辆物理极限.最高车速:
                        超限列表.append("车速超出车辆物理极限")
                        修正建议.车速 = 车辆物理极限.最高车速

            // 3b. 输出校验结果
            IF 超限列表为空:
                向 ad-mcc-01 返回校验通过放行信号
            ELSE:
                // 紧急模式下，仅拦截绝对物理极限超限，模式边界超限仅告警不拦截
                IF internal_state == STATE_EMERGENCY:
                    仅保留物理极限超限项 = 过滤模式边界超限(超限列表)
                    IF 仅保留物理极限超限项为空:
                        向 ad-mcc-01 返回校验通过放行信号（附带超限告警）
                    ELSE:
                        向 ad-mcc-01 返回校验拦截信号（超限列表 + 修正建议）
                ELSE:
                    向 ad-mcc-01 返回校验拦截信号（超限列表 + 修正建议）

        SLEEP 1ms  // 高频响应，满足指令校验时效


FUNCTION 获取路面修正系数(摩擦系数, 操作类型):
    IF 摩擦系数 >= 0.7:
        RETURN 1.0
    ELSE IF 摩擦系数 >= 0.4:
        RETURN IF 操作类型 == "制动" THEN 0.7 ELSE 0.8
    ELSE IF 摩擦系数 >= 0.2:
        RETURN IF 操作类型 == "制动" THEN 0.4 ELSE 0.5
    ELSE:
        RETURN IF 操作类型 == "制动" THEN 0.2 ELSE 0.3
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 车辆物理极限参数加载失败 | 使用编译期默认极限值，上报告警 | 参数重新加载成功 |
| 路面摩擦系数数据不可用 | 保守假设为湿滑路面（摩擦系数=0.4），上报告警 | 摩擦系数数据恢复 |
| 紧急模式下模式边界超限 | 仅告警不拦截，允许紧急避险操作突破常规模式边界 | 紧急模式解除 |
| 多项参数同时超限 | 全部列出，逐项给出修正建议，不因单项超限跳过其他校验 | — |
| 参数更新过程中收到操控指令 | 使用更新前的旧参数完成校验，保证校验连续性 | 参数更新完成 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 任何操控指令在下发至执行模块前，必须通过本模块的边界校验。此为编译期强制约束，不可绕过 |
| S-02 | 车辆出厂标定的绝对物理极限（最高车速、最大转角速率等）为不可逾越的红线，任何模式不得突破 |
| S-03 | 紧急模式仅放宽模式边界约束，绝对物理极限仍死守不破 |
| S-04 | 路面摩擦系数不可用时，必须保守假设为低附着路面，采用最严格的修正系数 |
| S-05 | 本模块不参与场景判断与驾驶决策，仅提供物理可行性校验 |
