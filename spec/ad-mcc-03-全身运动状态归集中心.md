# ad-mcc-03-全身运动状态归集中心 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-03 |
| 模块名称 | 全身运动状态归集中心 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 实时汇总各执行模块（转向、动力、制动、姿态、灯光、档位等）的协同进度、执行偏差、平衡状态与动作流畅度。整合为统一的“全身运动状态快照”，生成运动质量评估报告与体态异常标记，周期性上报 ECC 大脑及 MLNF-Mem 记忆中枢。为 ECC 提供运动执行层面的完整闭环反馈，支撑元认知、决策追溯与运动调优 |
| 依赖模块 | ad-mcc-04 至 ad-mcc-38（各执行模块，周期性上报执行状态与偏差）、ad-mcc-37（运动质量评估单元，提供运动质量得分与优化建议） |
| 被依赖模块 | ad-mcc-01（小脑总控调度核心，接收状态汇总用于调度决策）、ECC-12 资源调度模块（通过 CerebellumBus 接收全身运动状态快照）、MLNF-Mem 记忆中枢（通过 MemoryBus 接收运动状态快照用于经验沉淀） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常归集 | `NORMAL_GATHERING` | 所有执行模块在线，正常归集状态数据 | 系统初始化完成，各模块通信正常 |
| 部分模块离线 | `PARTIAL_OFFLINE` | 个别执行模块无响应或超时，使用最后有效值填充 | 任一执行模块连续3个周期无状态上报 |
| 严重降级归集 | `SEVERE_DEGRADED` | 多个关键执行模块离线，标记数据完整性不足 | 转向、制动等关键模块离线或通信总线异常 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断，暂停状态归集 | 收到紧急熔断指令 |

## 核心处理逻辑

```
FUNCTION motion_state_gathering_main_loop():
    // 状态标识常量
    STATE_NORMAL = NORMAL_GATHERING
    STATE_PARTIAL = PARTIAL_OFFLINE
    STATE_SEVERE = SEVERE_DEGRADED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化模块心跳跟踪表（模块ID → 最后心跳时间 + 在线状态）
    初始化快照序列号 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收各模块状态更新
        IF 收到转向执行状态:
            更新模块心跳表["转向"] = NOW()
            缓存转向状态数据

        IF 收到动力执行状态:
            更新模块心跳表["动力"] = NOW()
            缓存动力状态数据

        IF 收到制动执行状态:
            更新模块心跳表["制动"] = NOW()
            缓存制动状态数据

        IF 收到姿态执行状态:
            更新模块心跳表["姿态"] = NOW()
            缓存姿态状态数据

        IF 收到灯光执行状态:
            更新模块心跳表["灯光"] = NOW()
            缓存灯光状态数据

        IF 收到档位执行状态:
            更新模块心跳表["档位"] = NOW()
            缓存档位状态数据

        // 第3步：模块心跳超时检查
        FOR EACH 模块 IN 心跳跟踪表:
            IF NOW() - 模块.最后心跳时间 > 3 × 上报周期:
                标记模块为离线
                IF 模块 IN [转向, 制动]:
                    SET internal_state = STATE_SEVERE
                ELSE:
                    SET internal_state = STATE_PARTIAL
                // 使用最后有效值填充缺失数据
            ELSE:
                标记模块为在线

        IF 所有模块在线:
            SET internal_state = STATE_NORMAL

        // 第4步：周期性生成全身运动状态快照（每100ms）
        IF 距上次快照生成 >= 100ms:
            快照序列号++
            在线模块数 = 统计在线模块数()
            模块在线率 = 在线模块数 / 总模块数

            IF 模块在线率 >= 1.0:
                数据完整性 = "完整"
            ELSE IF 模块在线率 >= 0.7:
                数据完整性 = "部分缺失"
            ELSE:
                数据完整性 = "严重缺失"

            全身快照 = FullBodyMotionSnapshot{
                快照时间戳: NOW(),
                快照序列号: 快照序列号,
                转向状态: 缓存转向状态数据（若离线则用最后有效值并标记在线=False）,
                动力状态: ...,
                制动状态: ...,
                姿态状态: ...,
                灯光状态: ...,
                档位状态: ...,
                数据完整性: 数据完整性,
                模块在线率: 模块在线率
            }

            // 输出快照
            向 ad-mcc-01 发送全身运动状态快照
            向 ECC-12 发送全身运动状态快照（通过 CerebellumBus）
            向 MLNF-Mem 发送全身运动状态快照（通过 MemoryBus，供经验沉淀）

        // 第5步：接收运动质量评估报告并转发
        IF 收到 ad-mcc-37 运动质量评估报告:
            向 ECC-08 发送运动质量评估报告（通过 CerebellumBus）
            向 MLNF-Mem 发送运动质量评估报告（通过 MemoryBus）

        // 第6步：体态异常检测与标记
        IF 转向偏差 > 5° OR 速度偏差 > 3km/h OR 减速度偏差 > 20% OR 侧翻风险等级 >= 高:
            体态异常 = {
                异常类型: 判定异常类型(),
                异常模块: 判定异常模块(),
                严重等级: IF 侧翻风险 >= 高 OR 制动偏差 > 30% THEN "严重" ELSE "一般",
                建议动作: IF 严重等级 == "严重" THEN "触发降级或紧急停车" ELSE "提示关注"
            }
            向 ad-mcc-01 发送体态异常标记
            向 ECC-12 发送体态异常标记

        // 第7步：周期性模块在线状态上报
        IF 距上次在线状态上报 >= 1秒:
            向 ad-mcc-01 发送模块在线状态清单

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 执行模块超时无响应（>3个上报周期） | 标记该模块为离线，使用最后有效值填充缺失数据，上报告警 | 模块恢复上报 |
| 多个关键模块同时离线（转向/制动） | 标记数据完整性为“严重缺失”，触发严重降级归集状态，上报告警 | 模块恢复 |
| 运动质量评估报告超时未收到 | 使用上一周期报告，标记“评估报告可能过时” | 报告恢复接收 |
| 全身快照生成时部分数据缺失 | 缺失字段填充最后有效值并标记对应模块在线=False | 数据恢复 |
| 紧急熔断 | 暂停状态归集，保留最后有效快照 | 紧急解除后恢复 |

| CerebellumBus | 写 | 体态异常标记 | 事件触发写入 | 向 ECC-12 发送 |

| MemoryBus | 写 | 运动质量评估报告 | 专属写入 | 向 MLNF-Mem 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块仅做状态归集与上报，不参与任何操控决策。异常标记仅作为建议，不直接触发执行动作 |
| S-02 | 模块离线时，必须使用最后有效值填充缺失数据并明确标记在线=False，不可伪造在线状态 |
| S-03 | 全身运动状态快照中的数据须与各执行模块上报的原始数据保持一致，不得篡改或修饰 |
| S-04 | 数据完整性为“严重缺失”时，快照中须明确标注，供 ECC 决策时参考 |
