# ad-MCC-01-小脑总控调度核心 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-MCC-01 |
| 模块名称 | 小脑总控调度核心 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | MCC 运动小脑最高统筹单元。接收 ECC 大脑下发的标准化行驶意图指令，解析指令类型与优先级，统一分发至转向、动力、制动、姿态、灯光、档位等各执行集群。汇总各子模块的执行状态反馈，形成运动闭环回执上报 ECC。调度不同执行模式（正常/降级/紧急/非铺装）之间的平滑切换。不参与任何场景判断与驾驶决策 |
| 依赖模块 | ECC-16（意图下发接口单元，通过 CerebellumBus 下发标准化行驶意图指令） |
| 被依赖模块 | MCC-02（运动生理边界闸门，接收初步校验后的指令）、MCC-03（全身运动状态归集中心，接收状态汇总触发）、MCC-04 至 MCC-38（全部执行模块，接收调度指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常执行 | `NORMAL_EXEC` | 正常接收 ECC 指令并按优先级分发 | 系统初始化完成，CerebellumBus 通信正常 |
| 降级执行 | `DEGRADED_EXEC` | 执行降级模式下的受限指令集 | 收到一级或二级降级指令 |
| 紧急执行 | `EMERGENCY_EXEC` | 执行最高优先级的紧急避险指令 | 收到紧急制动、碰撞后响应或三级降级指令 |
| 非铺装执行 | `UNPAVED_EXEC` | 执行非铺装道路柔和操控参数 | 收到非铺装道路标记信号 |
| 指令解析中 | `PARSING` | 正在解析 ECC 下发的行驶意图指令 | 收到新的 CerebellumBus 消息 |
| 维护锁定 | `MAINTENANCE_LOCK` | 系统自检未通过，禁止执行任何操控指令 | 冷启动自检失败或关键硬件故障 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 标准化行驶意图指令 | Struct（完整 CerebellumBus 消息：msg_id + 优先级 + 意图类型 + 目标车道 + 目标速度 + 约束参数 + 灯光指令 + 校验和） | ECC-16 意图下发接口单元（通过 CerebellumBus） | ECC 决策输出后 | **高** |
| 紧急制动/碰撞后响应指令 | Struct（意图类型=紧急制动 + 目标减速度 + 目标速度=0 + 紧急标记=True） | ECC-14 标准化意图格式封装单元（通过 ECC-16 转发，CRITICAL 优先级） | 碰撞风险或碰撞事件触发时 | **最高** |
| 各执行模块状态上报 | Struct（模块ID + 执行状态 + 当前值 + 偏差量 + 故障码） | MCC-04 至 MCC-38 各执行模块 | 周期性，100Hz | **高** |
| 驾驶模式信号 | Enum（正常/一级降级/二级降级/三级降级/非铺装道路） | ECC-12 资源调度模块（通过 CerebellumBus） | 驾驶模式变更时 | **高** |
| 冷启动自检结果 | Struct（自检通过/失败 + 未通过系统清单） | ECC-45 冷启动全车自检编排单元（通过 CerebellumBus） | 系统上电自检完成后 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ECC-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 转向调度指令 | Struct（目标转角 + 转角速率上限 + 模式标记） | MCC-04 方向盘转角解算单元 | 解析出变道/转弯/避让意图时 | **高** |
| 动力调度指令 | Struct（目标速度/加速度 + 冲击度约束 + 模式标记） | MCC-09 油门开度解算单元 | 解析出巡航/加速意图时 | **高** |
| 制动调度指令 | Struct（目标减速度 + 制动类型 + 制动模式 + 模式标记） | MCC-13 制动压力解算单元 | 解析出制动/减速意图时 | **高** |
| 姿态调度指令 | Struct（期望横摆角速度 + 侧向加速度约束 + 模式标记） | MCC-18 车身姿态实时监测单元 | 变道或转弯意图时 | 普通 |
| 灯光调度指令 | Struct（转向灯/双闪/远近光 + 触发条件） | MCC-22 转向灯自动控制单元 等 | 根据意图类型和场景触发 | 普通 |
| 档位调度指令 | Struct（目标档位 + 切换条件） | MCC-26 档位切换管控单元 | 停车/起步意图时 | 普通 |
| 执行模式切换信号 | Struct（新模式 + 切换原因 + 参数集） | MCC-02、MCC-03 及全部执行模块 | 驾驶模式变更时 | **高** |
| 运动闭环回执 | Struct（指令ID + 执行结果汇总 + 偏差摘要 + 时间戳） | ECC-16（通过 CerebellumBus） | 每条指令执行完成后 | **高** |
| 状态汇总上报 | Struct（当前模式 + 各模块执行状态 + 在线模块数） | MCC-03 全身运动状态归集中心、ECC-12（通过 CerebellumBus） | 周期性，每10ms | 普通 |


## 核心处理逻辑

```
FUNCTION mcc_dispatch_main_loop():
    // 状态标识常量
    STATE_NORMAL = NORMAL_EXEC
    STATE_DEGRADED = DEGRADED_EXEC
    STATE_EMERGENCY = EMERGENCY_EXEC
    STATE_UNPAVED = UNPAVED_EXEC
    STATE_PARSING = PARSING
    STATE_LOCKED = MAINTENANCE_LOCK
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_LOCKED
    初始化指令队列（按优先级排序）
    初始化当前执行模式 = "正常"

    WHILE 系统运行中:
        // 第1步：最高优先级——紧急指令
        IF 收到紧急制动或碰撞后响应指令:
            SET internal_state = STATE_EMERGENCY
            清空指令队列
            立即向 MCC-13 下发制动调度指令（目标减速度=最大制动力，制动类型=紧急）
            向 MCC-23 下发灯光调度指令（双闪+刹车灯高频闪烁）
            等待执行完成，向 ECC-16 发送运动闭环回执
            CONTINUE

        // 第2步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第3步：冷启动自检结果处理
        IF 收到冷启动自检结果 AND internal_state == STATE_LOCKED:
            IF 自检结果.通过:
                SET internal_state = STATE_NORMAL
            ELSE:
                // 保持维护锁定，上报 ECC-12
                CONTINUE

        // 第4步：驾驶模式切换
        IF 收到驾驶模式信号:
            SWITCH 驾驶模式信号:
                CASE "正常":
                    SET internal_state = STATE_NORMAL
                    当前执行模式 = "正常"
                CASE "一级降级":
                CASE "二级降级":
                    SET internal_state = STATE_DEGRADED
                    当前执行模式 = "降级"
                CASE "三级降级":
                    SET internal_state = STATE_EMERGENCY
                    当前执行模式 = "紧急"
                CASE "非铺装道路":
                    SET internal_state = STATE_UNPAVED
                    当前执行模式 = "非铺装"
            // 广播执行模式切换信号至全部执行模块
            向 MCC-02 至 MCC-38 发送执行模式切换信号（含新参数集）

        // 第5步：接收 ECC 行驶意图指令
        IF 收到标准化行驶意图指令:
            SET internal_state = STATE_PARSING
            指令 = 输入.标准化行驶意图指令

            // 5a. 解析指令类型
            CASE 指令.意图类型 OF:
                "CRUISE":
                    // 匀速巡航
                    向 MCC-09 下发动力调度指令（目标速度 + 巡航策略）
                    向 MCC-04 下发转向调度指令（车道保持微调）
                    向 MCC-24 下发灯光调度指令（根据环境切换远近光）

                "LANE_CHANGE":
                    // 车道变换
                    向 MCC-22 下发转向灯指令（提前开启）
                    向 MCC-04 下发转向调度指令（目标转角序列 + 横向冲击度约束）
                    向 MCC-09 下发动力调度指令（匹配目标车道速度）
                    向 MCC-18 下发姿态调度指令（侧向加速度约束）

                "BRAKE":
                    // 制动
                    向 MCC-13 下发制动调度指令（目标减速度 + 制动类型）
                    向 MCC-22/23 下发灯光调度指令（刹车灯）
                    IF 指令.紧急标记:
                        向 MCC-23 下发双闪指令

                "TURN":
                    // 转弯
                    向 MCC-22 下发转向灯指令
                    向 MCC-04 下发转向调度指令（目标转角 + 曲率半径约束）
                    向 MCC-09 下发动力调度指令（目标速度 + 转弯限速）
                    向 MCC-18 下发姿态调度指令（横摆角速度约束）

                "PARK":
                    // 停车
                    向 MCC-13 下发制动调度指令（平顺刹停）
                    向 MCC-26 下发档位指令（P档）
                    向 MCC-27 下发电子驻车指令

                "CHARGING":
                    // 补能导航
                    向 MCC-09 下发动力调度指令（经济巡航速度）
                    向 MCC-24 下发灯光调度指令

            // 5b. 等待各模块执行反馈
            WAIT 各模块执行完成信号（超时=指令.执行时限）
            汇总执行结果

            // 5c. 生成运动闭环回执
            回执 = {
                指令ID: 指令.msg_id,
                执行结果: IF 全部成功 THEN "完成" ELSE "部分异常",
                偏差摘要: 汇总各模块偏差量,
                时间戳: NOW()
            }
            向 ECC-16 发送运动闭环回执

            SET internal_state = 根据当前执行模式恢复

        // 第6步：周期性状态汇总
        IF 距上次状态汇总 >= 10ms:
            汇总各执行模块状态
            向 MCC-03 发送状态汇总
            向 ECC-12 发送状态汇总上报

        SLEEP 2ms  // 高频响应，满足500Hz控制周期
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 指令队列积压超过10条 | 压缩队列：仅保留最新2帧，丢弃旧帧，上报告警 | 队列恢复正常 |
| 单执行模块超时无反馈 | 标记该模块为“离线”，使用最后有效值继续执行，上报告警 | 模块恢复通信 |
| 多个执行模块同时离线 | 触发紧急停车流程，上报 ECC-12 | 模块恢复 |
| 指令解析失败（消息格式异常） | 丢弃异常指令，记录安全日志，保留上一条有效指令继续执行 | 下一帧指令正常 |
| 维护锁定状态下收到操控指令 | 拒绝执行，返回错误码 | 自检通过解除锁定 |
| 紧急执行模式与正常指令冲突 | 紧急指令优先，正常指令排队等待或丢弃 | 紧急模式解除 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| CerebellumBus | 读 | 标准化行驶意图指令 | 只读 | ECC-16 发送 |
| CerebellumBus | 读 | 紧急制动/碰撞后响应指令 | 只读（最高优先级） | ECC-14/16 发送 |
| CerebellumBus | 读 | 驾驶模式信号 | 只读 | ECC-12 发送 |
| CerebellumBus | 读 | 冷启动自检结果 | 只读 | ECC-45 发送 |
| CerebellumBus | 读 | 全局调度指令 | 只读 | ECC-12 发送 |
| CerebellumBus | 写 | 运动闭环回执 | 专属写入 | 向 ECC-16 发送 |
| CerebellumBus | 写 | 状态汇总上报 | 周期性写入 | 向 ECC-12 发送 |
| 内部调度总线 | 写 | 转向/动力/制动/姿态/灯光/档位调度指令 | 专属写入 | 向各执行模块发送 |
| 内部调度总线 | 读 | 各执行模块状态上报 | 只读 | 各执行模块周期性上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | MCC-01 为 ECC 大脑下发操控指令的唯一入口，任何模块不得绕过本模块直接操控车辆硬件 |
| S-02 | 紧急制动或碰撞后响应指令为最高优先级，可立即中断当前任何正在执行的非安全指令 |
| S-03 | 冷启动自检未通过时，MCC-01 必须处于维护锁定状态，拒绝执行任何操控指令 |
| S-04 | 本模块不参与任何场景判断与驾驶决策，仅负责指令解析、分发与执行反馈汇总 |
| S-05 | 执行模式切换时，各模块参数集必须通过 MCC-02 运动生理边界闸门校验后方可生效 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M01-01 | `NORMAL_EXEC`，收到巡航指令 | ECC 巡航指令（目标速度=100km/h，目标车道=2） | 向 MCC-09 下发动力调度指令，向 MCC-04 下发转向调度指令 |
| TC-M01-02 | `NORMAL_EXEC`，收到紧急制动指令（CRITICAL） | ECC 紧急制动指令（目标速度=0，紧急标记=True） | 清空指令队列，向 MCC-13 下发制动指令，向 MCC-23 下发双闪指令 |
| TC-M01-03 | `MAINTENANCE_LOCK`，收到操控指令 | ECC 巡航指令 | 拒绝执行，返回错误码 |
| TC-M01-04 | `NORMAL_EXEC`，收到降级信号 | 驾驶模式信号=一级降级 | 广播执行模式切换信号，状态切换 `DEGRADED_EXEC` |
| TC-M01-05 | `NORMAL_EXEC`，收到紧急熔断 | 紧急熔断指令 | 状态切换 `SYSTEM_PAUSED` |
