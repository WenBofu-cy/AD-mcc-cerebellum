# ad-mcc-01-小脑总控调度核心 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-01 |
| 模块名称 | 小脑总控调度核心 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 作为 MCC 运动小脑的全局唯一调度中枢，接收 ECC 大脑通过 CerebellumBus 下发的标准化行驶意图指令，完成意图解析、优先级仲裁、执行模式判定与分发路由。协调各执行模块的协同工作，汇总各子模块执行状态与偏差报告，统一向 ECC 大脑回传运动闭环反馈。管理全局执行模式的切换协调，确保所有受控模块同步切换。不参与任何场景判断与驾驶决策 |
| 依赖模块 | ad-mcc-02（运动生理边界闸门，指令校验）、ad-mcc-03（执行模式切换单元，模式参数集）、ad-mcc-36（执行闭环反馈单元，汇总回执） |
| 被依赖模块 | ECC 大脑（通过 CerebellumBus，下发意图）、ad-mcc-02 至 ad-mcc-38（全部 MCC 内部模块统一调度） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 初始化 | `INIT` | 系统上电自检完成前，拒绝所有操控指令 | 冷启动/热启动，ad-mcc-45 发出系统就绪前 |
| 就绪空闲 | `IDLE` | 系统就绪，等待 ECC 下发行驶意图 | 自检通过，无待执行指令 |
| 正常调度 | `NORMAL_DISPATCH` | 接收、解析、分发操控指令至各执行模块 | 收到 ECC 行驶意图指令 |
| 降级调度 | `DEGRADED_DISPATCH` | 以降级约束参数分发指令，同步切换所有模块 | 收到降级模式信号 |
| 紧急调度 | `EMERGENCY_DISPATCH` | 仅执行紧急避险与安全停车指令，其他指令搁置 | 收到三级降级或紧急制动信号 |
| 非铺装调度 | `UNPAVED_DISPATCH` | 以非铺装柔和参数分发指令 | 收到非铺装道路模式信号 |
| 熔断 | `MELTDOWN` | 全局紧急熔断，拒绝所有操控指令，仅接收恢复信号 | 收到紧急熔断信号或 ad-mcc-05/ad-mcc-12 触发熔断 |

| 各模块偏差超限告警 | Struct（模块编号 + 偏差类型 + 偏差量 + 时间戳） | ad-mcc-07 / ad-mcc-12 / ad-mcc-16 | 偏差超阈值时 | **高** |
| 运动质量评估报告 | Struct（纵向冲击度RMS + 横向冲击度RMS + 偏差累积 + 综合得分） | ad-mcc-37 | 每次操控动作完成后 | 普通 |
| 硬件异常告警 | Struct（模块编号 + 异常类型 + 异常等级 + 故障码） | ad-mcc-28 至 ad-mcc-31 | 异常触发时 | **紧急** |
| 动作完成回执 | Struct（指令ID + 执行结果 + 完成时间戳） | ad-mcc-36 | 每次操控动作完成后 | **高** |
| 系统就绪信号 | Struct（自检通过标记 + 可用模块清单 + 降级模块清单） | ad-mcc-45（冷启动全车自检编排单元，通过内部总线） | 冷启动/热启动完成时 | **高** |

## ECC 行驶意图指令结构体

```
ECC_DRIVING_INTENT = {
    intent_id:        uint32,          // 唯一指令ID（ECC 生成，用于闭环追踪）
    intent_type:      enum,            // 巡航 / 避险控速 / 车道变换 / 路口通行 / 补能导航
    timestamp:        uint64,          // ECC 下发时间戳（UTC 微秒）
    priority:         uint8,           // 优先级 0=最低, 255=最高（急停/紧急制动=255）
    ttl_ms:           uint16,          // 指令有效期（毫秒），超时未执行则丢弃
    target_speed:     float,           // 目标车速 km/h（巡航/限速场景使用）
    target_accel:     float,           // 目标加速度 m/s²（正值=加速，负值=减速）
    max_brake_decel:  float,           // 最大允许制动减速度 m/s²（避险控速场景）
    target_lane:      int8,            // 目标车道编号（-1=向左变道, 0=保持, 1=向右变道）
    lateral_offset:   float,           // 横向偏移距离 m（避让障碍物场景）
    turn_curvature:   float,           // 转弯曲率半径 m（路口通行场景）
    stop_line_dist:   float,           // 距停止线距离 m（路口通行场景）
    navigation_target: {               // 补能导航场景
        target_type:  enum,            // 充电站 / 加油站 / 安全返航点
        target_route: [uint32],        // 全局路径路段ID序列
        energy_level: enum             // 能源预警等级（绿/黄/橙/红）
    },
    scenario_tag:     enum,            // 场景标签（高速巡航 / 城区路口 / 泊车低速 / 特殊环境 / 非铺装道路）
    style_profile:    enum,            // 驾驶风格（平顺舒适 / 标准通勤 / 高效通行）
    risk_level:       uint8,           // 风险等级 0-10（0=无风险, 10=最高风险）
    cause_label:      string           // 原因标签（如"人行横道行人优先"、"前方坑洼避让"）
}
```


## 核心处理逻辑

```
FUNCTION mcc01_dispatch_main_loop():
    // 状态常量
    STATE_INIT = INIT
    STATE_IDLE = IDLE
    STATE_NORMAL = NORMAL_DISPATCH
    STATE_DEGRADED = DEGRADED_DISPATCH
    STATE_EMERGENCY = EMERGENCY_DISPATCH
    STATE_UNPAVED = UNPAVED_DISPATCH
    STATE_MELTDOWN = MELTDOWN

    SET internal_state = STATE_INIT
    当前待执行指令队列 = []
    当前活跃指令ID = NONE

    // 第0步：等待系统就绪
    WAIT 收到系统就绪信号
    SET internal_state = STATE_IDLE

    WHILE 系统运行中:
        // ==================== 优先级1：紧急熔断/急停 ====================
        IF 收到急停/运动锁定指令（来自 ad-ecc-05，优先级=最高）:
            SET internal_state = STATE_MELTDOWN
            清空当前待执行指令队列
            向全部内部模块广播全局熔断信号
            收集各模块急停确认回执
            向 ad-ecc-05 返回急停执行回执
            CONTINUE  // 等待恢复信号

        IF 收到恢复信号 AND internal_state == STATE_MELTDOWN:
            SET internal_state = STATE_IDLE
            CONTINUE

        // ==================== 优先级2：模式切换 ====================
        IF 收到执行模式切换请求:
            SWITCH 新模式:
                CASE "三级降级":
                    SET internal_state = STATE_EMERGENCY
                    清空当前待执行指令队列（非安全类搁置）
                CASE "一级降级":
                CASE "二级降级":
                    SET internal_state = STATE_DEGRADED
                CASE "非铺装道路":
                    SET internal_state = STATE_UNPAVED
                CASE "正常":
                    SET internal_state = STATE_NORMAL
            // 向全部执行模块广播全局执行模式切换信号（含各模式参数集）
            广播全局执行模式切换信号(新模式, 各模式对应参数集)
            CONTINUE

        // ==================== 优先级3：接收 ECC 行驶意图 ====================
        IF 收到 ECC 标准化行驶意图指令:
            IF internal_state == STATE_INIT OR internal_state == STATE_MELTDOWN:
                向 ECC 返回拒绝回执（原因=当前状态不可执行）
                CONTINUE

            // 3a. 意图优先级仲裁
            IF 新指令.priority > 当前活跃指令.priority:
                IF 当前活跃指令 != NONE:
                    中断当前活跃指令，向执行模块下发暂停信号
                设置当前活跃指令 = 新指令
                当前待执行指令队列.insert_top(新指令)  // 插入队首优先执行
            ELSE:
                当前待执行指令队列.append(新指令)  // 按优先级插入队列

            // 3b. 意图解析与分发
            IF 当前活跃指令.ttl_ms 已过期:
                标记指令为过期，跳过执行
                向 ECC 返回超时回执
                CONTINUE

            // 3c. 将解析后的操控参数发送至 ad-mcc-02 进行边界校验
            待校验指令 = 解析为操控参数(当前活跃指令)
            向 ad-mcc-02 发送待校验指令

        // ==================== 优先级4：接收边界校验结果 ====================
        IF 收到 ad-mcc-02 校验结果:
            IF 校验类型 == "校验通过放行":
                // 按指令类型分解为各执行模块的具体操控指令
                SWITCH 当前活跃指令.intent_type:
                    CASE "巡航":
                        向 ad-mcc-09 下发油门控制指令
                        向 ad-mcc-18~21 下发姿态监测指令
                    CASE "避险控速":
                        向 ad-mcc-13 下发制动指令
                        向 ad-mcc-23 下发双闪/刹车灯指令
                    CASE "车道变换":
                        向 ad-mcc-04 下发转向指令
                        向 ad-mcc-22 下发转向灯指令
                    CASE "路口通行":
                        向 ad-mcc-04 下发转向指令（含曲率半径）
                        向 ad-mcc-13 下发制动指令（含停止线距离）
                    CASE "补能导航":
                        向 ad-mcc-09 下发巡航指令（含目标路径）

            ELSE IF 校验类型 == "校验拦截":
                // 将拦截原因与修正建议回传 ECC，等待 ECC 重新下发修正后的意图
                向 ECC 返回校验拦截回执(拦截原因, 修正建议)
                从当前待执行指令队列移除该指令
                CONTINUE

        // ==================== 优先级5：执行状态监控与异常处理 ====================
        IF 收到各模块偏差超限告警:
            汇总偏差告警，向 ECC 实时回传
            IF 偏差超限持续 > 3 秒:
                向 ECC 建议触发安全降级

        IF 收到硬件异常告警:
            识别告警等级
            IF 告警等级 >= L3:
                向 ECC 上报告警，建议禁止启动自动驾驶
            ELSE IF 告警等级 == L2:
                向 ECC 上报告警，建议触发一级降级
            ELSE:
                记录日志，静默监控

        // ==================== 优先级6：动作完成闭环 ====================
        IF 收到 ad-mcc-36 动作完成回执:
            汇总各模块偏差报告与质量评分
            向 ECC 返回动作闭环回执
            清除当前活跃指令
            从当前待执行指令队列取出下一条指令（若存在）

        SLEEP 1ms  // 1kHz 主循环频率
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 系统初始化未完成收到操控指令 | 拒绝执行，返回拒绝回执（原因=系统未就绪） | 收到系统就绪信号 |
| 指令TTL超时 | 标记过期，跳过执行，向ECC回传超时回执 | ECC重新下发新指令 |
| 指令校验被ad-mcc-02拦截 | 向ECC回传拦截原因与修正建议，等待ECC修正后重新下发 | ECC重新下发修正后指令 |
| 偏差超限持续3秒以上 | 向ECC建议触发安全降级 | 偏差恢复正常范围或系统降级 |
| 收到硬件L3级告警 | 立即向ECC上报，建议禁止启动自动驾驶 | 故障修复，告警清除 |
| 当前指令执行中被更高优先级指令打断 | 向被打断指令对应模块下发暂停信号，记录打断点；优先执行高优先级指令 | 高优指令执行完毕 |
| 各模块回执超时未收到 | 轮询重试3次，仍超时则标记该模块异常，上报告警 | 模块恢复响应 |
| 急停指令下发后模块未回执 | 连续下发3次，每次间隔100ms；仍未响应则记录故障 | 模块恢复响应 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块为 MCC 运动小脑的唯一调度入口，所有 ECC 行驶意图必须经本模块接收与分发，禁止任何模块直接接收 ECC 指令 |
| S-02 | 所有操控指令在下发执行模块前，必须经 ad-mcc-02（运动生理边界闸门）校验。此为编译期强制约束 |
| S-03 | 急停/运动锁定指令的优先级为最高（priority=255），可打断任何正在执行的指令 |
| S-04 | 熔断状态下，本模块仅接收恢复信号，拒绝一切操控指令 |
| S-05 | 本模块不参与场景判断与驾驶决策，仅执行指令解析、校验委托、分发与反馈汇总 |
| S-06 | 指令TTL超时强制丢弃，不得执行过期指令，杜绝延迟指令引发的安全隐患 |
| S-07 | 本模块为 CerebellumBus 的唯一对端，ECC 大脑与 MCC 小脑的所有通信均由本模块中转 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M01-01 | 系统就绪，`IDLE` | ECC 下发正常巡航指令（priority=100） | 解析为油门控制指令 → 送 ad-mcc-02 校验 → 校验通过 → 下发至 ad-mcc-09 |
| TC-M01-02 | 正在执行低优先级指令（priority=100） | ECC 下发高优先级避险指令（priority=200） | 中断当前指令 → 优先执行避险指令 |
| TC-M01-03 | 系统处于 `INIT` 状态 | ECC 下发任意指令 | 拒绝执行，返回拒绝回执（原因=系统未就绪） |
| TC-M01-04 | 正在执行指令 | 收到 ad-ecc-05 急停指令（priority=255） | 进入 `MELTDOWN` → 广播熔断信号 → 清空队列 → 收集回执 → 返回急停执行回执 |
