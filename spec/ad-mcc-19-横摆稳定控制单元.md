| 模块名称 | 横摆稳定控制单元 |
| 所属分区 | 五、车身姿态稳定 |
| 核心职责 | 基于 ad-mcc-18 提供的车身横摆角速度、侧向加速度及方向盘转角、车速等信号，实时判断车辆是否出现不足转向或过度转向趋势。在极限工况下（如高速过弯、紧急变道），主动请求 ESP 对单个或多个车轮施加制动干预，并向动力系统请求降低驱动扭矩，以产生附加横摆力矩，维持车身稳定。不参与常规工况下的行驶决策，仅在稳定性风险出现时激活，且所有干预指令均通过 ad-mcc-02 边界校验后方可执行 |

## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 车辆横摆运动在稳定范围内，无干预需求 | 横摆角速度偏差 < 阈值且侧偏角 < 阈值 |
| 不足转向干预 | `UNDERSTEER_CONTROL` | 车辆前轴侧偏角过大，实际横摆角速度低于期望值，需对内侧后轮施加制动并适当降低扭矩 | 不足转向指数 > 阈值 |
| 过度转向干预 | `OVERSTEER_CONTROL` | 车辆后轴侧偏角过大，实际横摆角速度高于期望值，需对外侧前轮施加制动并降低扭矩 | 过度转向指数 > 阈值 |
| 临界稳定 | `CRITICAL_STABILITY` | 车辆即将进入不可控漂移或侧滑，执行最大程度干预 | 横摆角速度偏差极大或侧偏角超过安全极限 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 横摆稳定控制模型

### 一、期望横摆角速度计算

基于线性二自由度车辆模型，在给定方向盘转角和车速下，车辆的期望横摆角速度 \(r_{des}\) 为：

```
r_des = (v / L) * δ / (1 + (v/v_ch)²)
其中:
  v     = 当前车速 (m/s)
  L     = 轴距 (m)
  δ     = 前轮转角 (rad)，由方向盘转角 / 转向传动比得到
  v_ch  = 特征车速 (m/s)，v_ch = sqrt((L² * Cf * Cr) / (m * (Lr*Cr - Lf*Cf)))
```

### 二、横摆角速度偏差

```
e_r = r_actual - r_des
```

当 e_r > 阈值（如 +3°/s），且持续一定时间，判定为**过度转向**；当 e_r < -阈值（如 -3°/s），判定为**不足转向**。

### 三、附加横摆力矩计算

采用 PID 控制或模型预测控制计算所需附加横摆力矩 ΔM_z，然后根据车辆状态将其转化为具体车轮的制动压力。

- **过度转向**：对外侧前轮施加制动，产生反向横摆力矩。
- **不足转向**：对内侧后轮施加制动，产生正向横摆力矩。
- **临界稳定**：同时制动多个车轮并强制降扭。

制动压力估算：

```
P_brake = ΔM_z * R_brake_factor / (track/2 * 制动增益)
```

### 四、控制阈值表

| 参数名称 | 预警阈值 | 干预阈值 | 临界阈值 |
|----------|:---:|:---:|:---:|
| 横摆角速度偏差 (°/s) | ±2 | ±3 | ±8 |
| 侧偏角 (°) | ±3 | ±5 | ±10 |
| 侧偏角变化率 (°/s) | ±5 | ±10 | ±20 |


## 核心处理逻辑

```
FUNCTION yaw_stability_control_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_UNDER = UNDERSTEER_CONTROL
    STATE_OVER = OVERSTEER_CONTROL
    STATE_CRITICAL = CRITICAL_STABILITY
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化 PID 控制器参数
    初始化历史值

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE

        // 第2步：获取输入
        姿态 = 获取 ad-mcc-18 姿态向量
        IF 姿态.数据置信度 < 0.5:
            维持上一周期状态，不执行新干预
            CONTINUE

        方向盘转角 = 获取方向盘转角()
        车速_kmh = 获取当前车速()
        轴距 = 获取轴距()
        轮距 = 获取轮距()

        // 第3步：计算期望横摆角速度
        车速_ms = 车速_kmh / 3.6
        前轮转角 = 方向盘转角 / 转向传动比
        r_des = 计算期望横摆角速度(车速_ms, 前轮转角, 轴距)

        // 第4步：计算偏差
        r_actual = 姿态.横摆角速度_deg_per_s
        e_r = r_actual - r_des

        // 第5步：稳定性判定
        IF ABS(e_r) > 8.0 OR 侧偏角 > 10.0:
            SET internal_state = STATE_CRITICAL
            干预模式 = "临界稳定干预"
            降扭比例 = 0.5
        ELSE IF e_r > 3.0 AND 持续超过 50ms:
            SET internal_state = STATE_OVER
            干预模式 = "过度转向干预"
            降扭比例 = 0.3
        ELSE IF e_r < -3.0 AND 持续超过 50ms:
            SET internal_state = STATE_UNDER
            干预模式 = "不足转向干预"
            降扭比例 = 0.2
        ELSE:
            // 偏差回落到正常范围
            IF internal_state != STATE_NORMAL:
                记录干预结束事件
                SET internal_state = STATE_NORMAL
            释放所有制动干预
            降扭比例 = 0.0

        // 第6步：计算附加横摆力矩
        IF internal_state != STATE_NORMAL:
            ΔM_z = PID控制(e_r, de_r/dt)
            制动压力 = 计算制动压力(ΔM_z, 轮距, 干预模式)

            // 第7步：下发干预指令
            选择目标车轮 = 根据干预模式确定(前轮/后轮, 内/外侧)
            ESP指令 = 构建ESP制动指令(目标车轮, 制动压力, 干预优先级)
            降扭请求 = 构建降扭请求(降扭比例, 干预模式)

            向 ESP 执行器发送 ESP 制动干预指令
            向动力控制器发送动力降扭请求
            向 ad-mcc-01、ad-mcc-20 上报横摆稳定状态

            // 第8步：记录干预事件
            向 ad-mcc-38 发送干预事件记录

        SLEEP 5ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 姿态数据置信度低（<0.5） | 维持上一周期状态，不执行新干预，但现有干预继续维持 | 置信度恢复 |
| 制动系统最大干预压力达到上限 | 限制输出压力至上限值，同时加大降扭请求 | 制动压力需求回落 |
| 驾驶员主动反向打方向盘 | 识别驾驶员意图，抑制或减弱 ESP 干预，优先尊重驾驶员操作 | 驾驶员操作结束且车辆恢复稳定 |
| 侧翻风险同时触发（ad-mcc-20） | 侧翻保护优先级高于横摆控制，若 ad-mcc-20 要求制动，则协调制动分配 | 侧翻风险解除 |
| 紧急熔断 | 立即释放所有 ESP 干预，降扭请求归零 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | ESP 制动干预指令不得超过制动系统当前最大允许压力，防止管路过载 |
| S-02 | 当 ad-mcc-20 侧翻临界保护激活时，侧翻保护具有更高优先级，本模块需协调制动分配 |
| S-03 | 驾驶员主动反向操作时，应尊重驾驶员意图，减少或中止 ESP 干预 |
| S-04 | 本模块仅负责稳定性干预，不参与常规制动或驱动控制 |
