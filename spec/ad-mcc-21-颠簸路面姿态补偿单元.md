# ad-mcc-21-颠簸路面姿态补偿单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-21 |
| 模块名称 | 颠簸路面姿态补偿单元 |
| 所属分区 | 五、车身姿态稳定 |
| 核心职责 | 在非铺装颠簸路面（如乡村土路、碎石路、有明显车辙/坑洼路段）行驶时，基于 ad-mcc-18 提供的俯仰角、垂向加速度及世界模型提供的路面高度异常标注，主动调整悬架阻尼特性（若配备可调悬架），柔化车身晃动幅度与冲击感。同时向 ad-mcc-01 建议降低车速上限，并协调 ad-mcc-09（油门）与 ad-mcc-13（制动）采用柔和操控参数，减少因路面激励引起的车身俯仰与垂向振动。不参与常规铺装路面的行驶决策，仅在检测到持续颠簸时激活 |
| 依赖模块 | ad-mcc-18（车身姿态实时监测单元，提供俯仰角、垂向加速度）、ad-44（独立世界模型库，提供路面高度异常标注与路面类型）、ad-mcc-32（车辆尺寸参数管理单元，提供轴距、悬架行程参数）、当前车速（CAN 总线） |
| 被依赖模块 | ad-mcc-01（小脑总控调度核心，接收姿态补偿状态与限速建议）、ad-mcc-09（油门开度解算单元，参考柔和参数）、ad-mcc-13（制动压力解算单元，参考柔和参数）、悬架控制器（通过 CAN 总线或内部总线发送阻尼调整指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 铺装路面或颠簸指数低于阈值，悬架保持标准模式 | 系统初始化完成，路面平整 |
| 轻度颠簸 | `MILD_BUMP` | 检测到间歇性轻度颠簸，悬架适度柔化，车速建议略降 | 俯仰角波动幅度 > 2° 或垂向加速度 RMS > 0.3g，持续 3 秒 |
| 重度颠簸 | `SEVERE_BUMP` | 检测到持续性重度颠簸，悬架最大柔化，车速强制限制 | 俯仰角波动幅度 > 5° 或垂向加速度 RMS > 0.6g，持续 2 秒 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 车身姿态状态向量 | Struct（俯仰角_deg + 垂向加速度_ms2 + 纵向加速度_ms2 + 数据置信度） | ad-mcc-18 车身姿态实时监测单元 | 每周期（200Hz） | **高** |
| 路面高度异常标注 | Struct（路面类型 + 颠簸指数 + 坑洼深度列表 + 车辙深度） | ad-44 独立世界模型库（通过 MemoryBus） | 周期性，1Hz 或事件触发 | **高** |
| 当前车速 | Float（km/h） | CAN 总线（轮速传感器） | 实时，100Hz | **高** |
| 悬架特性参数 | Struct（悬架类型 + 阻尼调节范围 + 最大悬架行程_mm + 响应时间_ms） | ad-mcc-32 车辆尺寸参数管理单元 | 系统初始化加载 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ad-mcc-01 小脑总控调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 悬架阻尼调整指令 | Struct（目标阻尼模式 + 前轴阻尼系数 + 后轴阻尼系数 + 调整原因 + 期望响应时间） | 悬架控制器（通过 CAN 总线或内部总线） | 颠簸状态变更时 | **高** |
| 车速上限建议 | Struct（建议车速上限_kmh + 建议原因 + 颠簸等级） | ad-mcc-01 小脑总控调度核心 | 颠簸状态变更或周期性（每1秒） | 普通 |
| 柔和操控参数建议 | Struct（建议加速度上限 + 建议减速度上限 + 建议冲击度上限） | ad-mcc-01（转发至 ad-mcc-09、ad-mcc-13） | 颠簸状态变更时 | 普通 |

| 颠簸补偿事件记录 | Struct（触发原因 + 补偿等级 + 执行动作 + 持续时间） | ad-mcc-38 执行日志记录单元 | 每次补偿状态变更时 | 普通 |


## 颠簸判定与补偿策略

### 一、颠簸指数计算

基于俯仰角波动幅度（连续 N 帧的峰峰值）和垂向加速度 RMS 值综合判定：

```
俯仰波动 = max(俯仰角) - min(俯仰角)  在滑动窗口 1 秒内
垂向RMS = sqrt(mean(垂向加速度²))     在滑动窗口 1 秒内

颠簸指数 = 0.5 × (俯仰波动 / 2°) + 0.5 × (垂向RMS / 0.3g)
```

### 二、分级补偿策略

| 补偿等级 | 触发条件 | 悬架阻尼模式 | 建议车速上限 | 建议加速度上限 | 建议减速度上限 |
|----------|----------|:---:|:---:|:---:|:---:|
| 轻度颠簸 | 颠簸指数 ≥ 1.0 且 < 2.0 | 舒适模式（阻尼降低 30%） | 当前车速（不强制） | 正常值的 80% | 正常值的 70% |
| 重度颠簸 | 颠簸指数 ≥ 2.0 | 越野模式（阻尼降至最低） | 25 km/h | 正常值的 50% | 正常值的 40% |

### 三、悬架阻尼控制逻辑

若车辆配备可调阻尼悬架（CDC/空气悬架）：
- **舒适模式**：压缩阻尼降低 30%，回弹阻尼降低 20%，减少路面冲击传递。
- **越野模式**：压缩阻尼降至最低，回弹阻尼适当提高（防止悬架触底反弹），最大化车轮贴地性。


## 核心处理逻辑

```
FUNCTION bump_compensation_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_MILD = MILD_BUMP
    STATE_SEVERE = SEVERE_BUMP
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化滑动窗口（俯仰角历史、垂向加速度历史）
    初始化颠簸指数 = 0.0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            恢复悬架标准模式
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：获取输入
        姿态 = 获取 ad-mcc-18 姿态向量
        IF 姿态.数据置信度 < 0.5:
            CONTINUE

        路面标注 = 获取 ad-mcc-44 路面高度异常标注
        当前车速 = 获取当前车速()
        悬架参数 = 获取悬架特性参数()

        // 第3步：更新滑动窗口
        俯仰窗口.append(姿态.俯仰角_deg)
        垂向窗口.append(姿态.垂向加速度_ms2)
        保持窗口大小 = 1秒（200帧）

        // 第4步：计算颠簸指数
        俯仰波动 = max(俯仰窗口) - min(俯仰窗口)
        垂向RMS = sqrt(mean(垂向窗口²))
        颠簸指数 = 0.5 × (俯仰波动 / 2.0) + 0.5 × (垂向RMS / (0.3 × 9.81))

        // 第5步：状态判定
        previous_state = internal_state

        IF 颠簸指数 >= 2.0:
            SET internal_state = STATE_SEVERE
        ELSE IF 颠簸指数 >= 1.0:
            IF internal_state != STATE_SEVERE:
                SET internal_state = STATE_MILD
        ELSE:
            SET internal_state = STATE_NORMAL

        // 第6步：执行补偿动作
        IF internal_state == STATE_SEVERE:
            悬架模式 = "越野模式"
            阻尼指令 = {前轴=最低, 后轴=最低+回弹提高}
            车速上限 = 25
            加速度上限 = 正常值 × 0.5
            减速度上限 = 正常值 × 0.4
        ELSE IF internal_state == STATE_MILD:
            悬架模式 = "舒适模式"
            阻尼指令 = {前轴=标准×0.7, 后轴=标准×0.7}
            车速上限 = 当前车速
            加速度上限 = 正常值 × 0.8
            减速度上限 = 正常值 × 0.7
        ELSE:
            悬架模式 = "标准模式"
            阻尼指令 = {前轴=标准, 后轴=标准}
            车速上限 = None
            加速度上限 = 正常值
            减速度上限 = 正常值

        // 第7步：下发指令
        IF internal_state != STATE_NORMAL:
            向悬架控制器发送悬架阻尼调整指令(悬架模式, 阻尼指令)
            IF 车速上限 != None:
                向 ad-mcc-01 发送车速上限建议(车速上限, 颠簸等级)
            向 ad-mcc-01 发送柔和操控参数建议(加速度上限, 减速度上限)

        // 第8步：恢复标准模式
        IF internal_state == STATE_NORMAL AND previous_state != STATE_NORMAL:
            向悬架控制器发送悬架阻尼调整指令("标准模式", 标准阻尼)
            向 ad-mcc-01 发送取消车速限制

        // 第9步：周期性上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 上报颠簸补偿状态(
                当前状态=internal_state,
                俯仰角RMS=俯仰波动,
                垂向加速度RMS=垂向RMS,
                当前阻尼模式=悬架模式,
                建议车速=车速上限
            )

        // 第10步：记录事件
        IF internal_state != previous_state:
            向 ad-mcc-38 记录颠簸补偿事件

        SLEEP 5ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 姿态数据置信度低（<0.5） | 维持当前补偿状态，不切换悬架模式 | 置信度恢复 |
| 车辆未配备可调悬架 | 跳过悬架阻尼调整，仅输出车速上限建议与柔和操控参数 | — |
| 路面标注数据不可用 | 仅依赖姿态数据进行颠簸判定，悬架保持当前模式 | 数据恢复 |
| 颠簸中驾驶员急加速 | 柔和操控参数建议保持有效，但不强制干预驾驶员操作 | 急加速结束 |
| 紧急制动触发 | 颠簸补偿不影响制动性能，悬架保持当前模式 | 紧急制动解除 |
| 紧急熔断 | 恢复悬架标准模式，取消所有限制 | 紧急解除后恢复 |

| 内部调度总线 | 写 | 柔和操控参数建议 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 颠簸补偿状态上报 | 周期性写入 | 向 ad-mcc-01 发送 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块仅建议限速与柔和参数，不直接控制油门、制动或转向执行器 |
| S-02 | 紧急制动时，颠簸补偿不得干预制动力分配或制动压力建立 |
| S-03 | 若车辆未配备可调悬架，不得输出悬架阻尼调整指令 |
| S-04 | 颠簸补偿的车速上限建议不得低于 20 km/h，确保车辆仍具备基本通行能力 |

| 用例编号 | 前置条件 | 输入 | 预期输出 |

| TC-M21-04 | `SEVERE_BUMP`，颠簸减弱 | 颠簸指数降至0.8 | 恢复 `NORMAL_MONITOR`，悬架切回标准模式 |
