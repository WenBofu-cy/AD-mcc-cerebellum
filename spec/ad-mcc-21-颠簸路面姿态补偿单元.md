# ad-mcc-21-颠簸路面姿态补偿单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-mcc-21 |
| 模块名称 | 颠簸路面姿态补偿单元 |
| 所属分区 | 五、车身姿态稳定 |
| 核心职责 | 在非铺装颠簸路面行驶时，基于 ad-mcc-18 提供的俯仰角、垂向加速度及世界模型提供的路面高度异常标注，主动调整悬架阻尼特性（若配备可调悬架），柔化车身晃动幅度与冲击感。同时向 ad-mcc-01 建议降低车速上限，并协调 ad-mcc-09（油门）与 ad-mcc-13（制动）采用柔和操控参数，减少因路面激励引起的车身俯仰与垂向振动。不参与常规铺装路面的行驶决策，仅在检测到持续颠簸时激活 |
| 依赖模块 | ad-mcc-18（车身姿态实时监测单元，提供俯仰角、垂向加速度）、ad-44（独立世界模型库，提供路面高度异常标注与路面类型）、ad-mcc-32（车辆尺寸参数管理单元，提供轴距、悬架行程参数）、当前车速（CAN 总线） |
| 被依赖模块 | ad-mcc-01（小脑总控调度核心，接收姿态补偿状态与限速建议）、ad-mcc-09（油门开度解算单元，参考柔和参数）、ad-mcc-13（制动压力解算单元，参考柔和参数）、悬架控制器（通过 CAN 总线或内部总线发送阻尼调整指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 铺装路面或颠簸指数低于阈值，悬架保持标准模式 | 系统初始化完成，路面平整 |
| 轻度颠簸 | `MILD_BUMP` | 检测到间歇性轻度颠簸，悬架适度柔化，车速建议略降 | 俯仰角波动幅度 > 2° 或垂向加速度 RMS > 0.3g，持续 3 秒 |
| 重度颠簸 | `SEVERE_BUMP` | 检测到持续性重度颠簸，悬架最大柔化，车速强制限制 | 俯仰角波动幅度 > 5° 或垂向加速度 RMS > 0.6g，持续 2 秒 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

| 悬架阻尼调整指令 | Struct（目标阻尼模式 + 前轴阻尼系数 + 后轴阻尼系数 + 调整原因 + 期望响应时间） | 悬架控制器（通过 CAN 总线或内部总线） | 颠簸状态变更时 | **高** |

## 颠簸判定与补偿策略

### 一、颠簸指数计算

基于俯仰角波动幅度（连续 200 帧的峰峰值）和垂向加速度 RMS 值综合判定：

```
俯仰波动 = max(俯仰窗口) - min(俯仰窗口)  滑动窗口 1 秒（200帧）
垂向RMS = sqrt(mean(垂向窗口²))           滑动窗口 1 秒

颠簸指数 = 0.5 × (俯仰波动 / 2°) + 0.5 × (垂向RMS / 0.3g)
其中 0.3g ≈ 0.3 × 9.81 = 2.943 m/s²
```

### 二、分级补偿策略

| 补偿等级 | 触发条件 | 悬架阻尼模式 | 建议车速上限 | 建议加速度上限 | 建议减速度上限 |
|----------|----------|:---:|:---:|:---:|:---:|
| 轻度颠簸 | 颠簸指数 ≥ 1.0 且 < 2.0 | 舒适模式（压缩阻尼 70%、回弹阻尼 80%） | 当前车速（不强制降速） | 正常值 × 0.8 | 正常值 × 0.7 |
| 重度颠簸 | 颠簸指数 ≥ 2.0 | 越野模式（压缩阻尼 50%、回弹阻尼 70%） | 25 km/h | 正常值 × 0.5 | 正常值 × 0.4 |

### 三、悬架阻尼控制

若车辆配备可调阻尼悬架（CDC/空气悬架），下发阻尼调整指令；若无，则跳过悬架调整，仅输出车速上限与柔和参数建议。

- **舒适模式**：前轴阻尼系数 0.7，后轴阻尼系数 0.7。
- **越野模式**：前轴阻尼系数 0.5，后轴阻尼系数 0.7（回弹略高防触底）。
- **标准模式**：前后轴阻尼系数均为 1.0。


## 核心处理逻辑

```
FUNCTION bump_compensation_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_MILD = MILD_BUMP
    STATE_SEVERE = SEVERE_BUMP
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    SET prev_state = STATE_NORMAL
    初始化俯仰滑动窗口 = 空双端队列（最大200帧）
    初始化垂向滑动窗口 = 空双端队列（最大200帧）
    SET has_adjustable_suspension = False
    SET normal_accel = 3.0, normal_decel = 5.0, normal_jerk = 5.0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            恢复悬架标准模式()
            取消所有限速与柔和参数建议()
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：获取输入
        姿态 = 获取 ad-mcc-18 姿态向量
        IF 姿态.数据置信度 < 0.5:
            CONTINUE

        路面标注 = 获取 ad-mcc-44 路面高度异常标注()
        车速 = 获取当前车速()
        悬架参数 = 获取悬架特性参数()
        IF 悬架参数.悬架类型 包含 "CDC" OR "空气":
            has_adjustable_suspension = True
        ELSE:
            has_adjustable_suspension = False

        // 第3步：更新滑动窗口
        俯仰窗口.append(姿态.俯仰角_deg)
        垂向窗口.append(姿态.垂向加速度_ms2)

        IF 俯仰窗口.size < 200:
            CONTINUE  // 窗口未满，等待数据积累

        // 第4步：计算颠簸指数
        俯仰波动 = max(俯仰窗口) - min(俯仰窗口)
        垂向RMS = sqrt(mean(垂向窗口各值²))
        颠簸指数 = 0.5 × (俯仰波动 / 2.0) + 0.5 × (垂向RMS / 2.943)

        // 第5步：状态判定
        prev_state = internal_state
        IF 颠簸指数 >= 2.0:
            SET internal_state = STATE_SEVERE
        ELSE IF 颠簸指数 >= 1.0:
            SET internal_state = STATE_MILD
        ELSE:
            SET internal_state = STATE_NORMAL

        // 第6步：执行补偿动作
        IF internal_state == STATE_SEVERE:
            IF has_adjustable_suspension:
                向悬架控制器发送阻尼指令("越野模式", 前轴=0.5, 后轴=0.7)
            向 ad-mcc-01 发送车速上限建议(25 km/h, "重度颠簸", 等级=2)
            向 ad-mcc-01 发送柔和操控参数(加速度=1.5, 减速度=2.0, 冲击度=2.5)

        ELSE IF internal_state == STATE_MILD:
            IF has_adjustable_suspension:
                向悬架控制器发送阻尼指令("舒适模式", 前轴=0.7, 后轴=0.7)
            向 ad-mcc-01 发送车速上限建议(车速, "轻度颠簸", 等级=1)
            向 ad-mcc-01 发送柔和操控参数(加速度=2.4, 减速度=3.5, 冲击度=4.0)

        ELSE: // STATE_NORMAL
            IF prev_state != STATE_NORMAL:
                IF has_adjustable_suspension:
                    向悬架控制器发送阻尼指令("标准模式", 前轴=1.0, 后轴=1.0)
                向 ad-mcc-01 发送取消限速()
                向 ad-mcc-01 发送柔和操控参数(加速度=3.0, 减速度=5.0, 冲击度=5.0)

        // 第7步：周期性上报
        IF 距上次上报 >= 1秒:
            向 ad-mcc-01 上报颠簸补偿状态(当前状态, 俯仰波动, 垂向RMS, 阻尼模式, 建议车速)

        // 第8步：状态变更记录
        IF internal_state != prev_state:
            向 ad-mcc-38 记录颠簸补偿事件(前状态, 新状态, 颠簸指数)

        SLEEP 5ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 姿态数据置信度 < 0.5 | 维持当前补偿状态，不切换悬架模式，窗口数据不更新 | 置信度恢复 ≥ 0.5 |
| 车辆未配备可调悬架 | 跳过悬架阻尼调整指令，仅输出车速上限建议与柔和操控参数建议 | — |
| 路面标注数据不可用 | 仅依赖姿态数据进行颠簸判定，悬架保持当前模式 | 路面数据恢复 |
| 颠簸中驾驶员急加速 | 柔和操控参数建议保持有效，但不强制干预驾驶员操作 | 急加速结束 |
| 紧急制动触发 | 颠簸补偿不影响制动性能，悬架保持当前模式，不输出柔和制动参数 | 紧急制动解除 |
| 紧急熔断 | 恢复悬架标准模式，取消所有限速与柔和参数建议 | 紧急解除后恢复 |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |

| MemoryBus | 读 | 路面高度异常标注 | 只读 | ad-44 提供 |
| CAN 总线 | 读 | 当前车速 | 只读 | 实时 |

| CAN 总线/内部总线 | 写 | 悬架阻尼调整指令 | 专属写入 | 向悬架控制器发送 |
| 内部调度总线 | 写 | 车速上限建议 | 周期性写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 柔和操控参数建议 | 事件触发写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 颠簸补偿状态上报 | 周期性写入 | 向 ad-mcc-01 发送 |
| 内部调度总线 | 写 | 颠簸补偿事件记录 | 事件触发写入 | 向 ad-mcc-38 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块仅建议限速与柔和参数，不直接控制油门、制动或转向执行器 |
| S-02 | 紧急制动时，颠簸补偿不得干预制动力分配或制动压力建立 |
| S-03 | 若车辆未配备可调悬架，不得输出悬架阻尼调整指令 |
| S-04 | 颠簸补偿的车速上限建议不得低于 20 km/h，确保车辆仍具备基本通行能力 |
